**Introduction**\n**LangChain is a framework that enables quick and easy development of applications that make use of Large Language Models**, for example, GPT-3.\nThe framework, however, introduces additional possibilities, for example, the one of easily using external data sources, such as Wikipedia, to amplify the capabilities provided by the model. I am sure that you have all probably tried to use Chat-GPT and find that it fails to answer about events that occurred beyond a certain date. In this case, a search on Wikipedia could help GPT to answer more questions.\n# **LangChain Structure**\nThe framework is organized into six modules each module allows you to manage a different aspect of the interaction with the LLM. Let’s see what the modules are.\n- [**Models**](https://python.langchain.com/en/latest/modules/models.html): Allows you to instantiate and use different models.\n- [**Prompts**](https://python.langchain.com/en/latest/modules/prompts.html): The prompt is how we interact with the model to try to obtain an output from it. By now knowing how to write an effective prompt is of critical importance. This framework module allows us to better manage prompts. For example, by creating templates that we can reuse.\n- [**Indexes**](https://python.langchain.com/en/latest/modules/indexes.html): The best models are often those that are combined with some of your textual data, in order to add context or explain something to the model. This module helps us do just that.\n- [**Chains**](https://python.langchain.com/en/latest/modules/chains.html): Many times to solve tasks a single API call to an LLM is not enough. This module allows other tools to be integrated. For example, one call can be a composed chain with the purpose of getting information from Wikipedia and then giving this information as input to the model. This module allows multiple tools to be concatenated in order to solve complex tasks.\n- [**Memory**](https://python.langchain.com/en/latest/modules/memory.html): This module allows us to create a persisting state between calls of a model. Being able to use a model that remembers what has been said in the past will surely improve our application.\n- [**Agents**](https://python.langchain.com/en/latest/modules/agents.html): An agent is an LLM that makes a decision, takes an action, makes an observation about what it has done, and continues in this manner until it can complete its task. This module provides a set of agents that can be used.\nNow let’s go into a little more detail and see how to implement code by taking advantage of the different modules.\n# **Models**\n*Models *allows the use of three different types of language-models, which are:\n- **Large Language Models (LLMs):** these foundational machine learning models that are able to understand natural language. These accept strings in input and generate strings in output.\n- **Chat Models: **models powered by LLM but are specialized to chat with the user. You can read more [here](https://medium.com/aiguys/reinforcement-learning-from-human-feedback-instructgpt-and-chatgpt-693d00cb9c58).\n- **Text Embedding Models: **these models are used to project textual data into a geometric space. These models take text as input and return a list of numbers, the embedding of the text.\n![](https://miro.medium.com/v2/resize:fit:700/1*_2RdfRviIL_hEeTMMIbP2g.png)\nOpen AI API Key\nLet’s start using this module. First, we install and import the libraries. To use this library you will need an API KEY from the open AI site.\n```plain text\n!pip install langchain >> null\n!pip install openai >> null\n```\n```python\nfrom langchain.llms import OpenAI\n```\n```python\n#past you api key here\n\nimport os\nos.environ['OPENAI_API_KEY'] = \"yuor-openai-key\"\n```\nNow we are ready to instantiate a LLM model.\n```python\nllm = OpenAI(model_name=\"text-ada-001\", n=2, best_of=2)\n```\n```python\nllm(\"tell me a story please.\")\n```\n***My Output**:* *A young woman went to a rave she had never heard of. She was the only person there with a light in the dark, and the only one who could see the good in people. She drank and danced and did anything to make it so. After hour after hour of dancing and drinkin', she got to know one of the people in the party. He was a bit of a looking man, with a moustache and a goatee. She said, \"Hi, I'm the only person there with a light in the dark.\" He said, \"Hi, I'm the only person there with a light in the dark.*\nWith the generate() method you can also feed a list of input and receive multiple answers, let’s see how.\n```python\nllm_result = llm.generate([\"Tell me a short story\", \"whath is your favourite colour?\", \"Is the earth flat?\"])\n```\n![](https://miro.medium.com/v2/resize:fit:700/1*KvvoTSGcDP6zVW5Oc5SDnQ.png)\nllm.generate output\nYou can also extract some additional information about the results of the large language model.\n```python\nllm_result.llm_output\n```\n***My Output:** \\{‘token_usage’: \\{‘completion_tokens’: 527, ‘total_tokens’: 544, ‘prompt_tokens’: 17\\}, ‘model_name’: ‘text-ada-001’\\}*\nLLMs are not able to understand input texts that are too long. In particular, text that contains too many tokens (think about the syllables of words if you don’t know what tokens are). Before passing the string into the model you can estimate the number of tokens with a simple method.\nIn order to do this you need to install the [tiktoken](https://github.com/openai/tiktoken) library.\n```python\n!pip install tiktoken >> None\nimport tiktoken\n```\n```python\nllm.get_num_tokens(\"How many old are you?\")\n\n#OUTPUT : 6\n```\n# **Prompts**\nPrompting is the new way of programming NLP models. Creating a good prompt though is not trivial. Asking the same thing in a different way can lead to a different result that is more or less accurate. The prompt can also change depending on the use case you are facing. Let’s see how this module can help us create a good prompt.\n## **Prompt Template**\nAs the name implies, the prompt template allows us to create templates that we can reuse to ask things of our model. The template will contain variables that will be the only thing that the user will change from time to time to adapt the prompt to its particular use case.\nLet us now see how they can be used.\n```python\nfrom langchain import PromptTemplate\n\ntemplate = \"\"\"\nI want you to act as businessman.\n\nYou have few passions in your life which are:\n\n- Money\n- Data\n- Basketball\n\nI want to write a Medium Blog post about {product}.\nWhat is a good for a title of such post?\n\"\"\"\n\nprompt = PromptTemplate(\n    input_variables=[\"product\"],\n    template=template,\n)\n```\nNow we can replace the variable ‘product’ within the prompt with whatever string we want. This way we can customize the prompt any way we like.\n```python\nprompt.format(product = \"how to make a bunch of money\")\n```\n***My Outupt: **I want you to act as businessman. You have few passions in your life which are:- Money- Data- Basketball. I want to write a Medium Blog post about how to make a bucnh of money. What is a good for a title of such post?*\nThere are templates that have already been written and you can simply import them. To find out what they are you can look at the documentation. Let’s try to import one now.\n```python\nfrom langchain.prompts import load_prompt\n\nprompt = load_prompt(\"lc://prompts/conversation/prompt.json\")\nprompt\n```\n***My Outupt: **PromptTemplate(input_variables=\\[‘history’, ‘input’\\], output_parser=None, partial_variables=\\{\\}, template=’The following is a friendly conversation between a human and an AI. The AI is talkative and provides lots of specific details from its context. If the AI does not know the answer to a question, it truthfully says it does not know.\\\\n\\\\nCurrent conversation:\\\\n\\{history\\}\\\\nHuman: \\{input\\}\\\\nAI:’, template_format=’f-string’, validate_template=True)*\nIn this template, we have multiple variables that we can fill in. One of them is *history*. We need the history to tell the template about something that happened previously so that it has more context. If we want to request the output of the model from this template, it is very simple.\n```python\nllm(prompt.format(history=\"\", input=\"What is 3 - 3?\"))\n```\n## **Few shot examples**\nSometimes it happens that we want to question the Machine Learning model about things that are particularly tricky. One way to get a more accurate answer, in this case, is to show the model similar examples of the correct answer and then ask it our question. LangChain provides a method for saving templates that are designed specifically to save these examples. Let’s see how to do this.\nFirst of all, let’s create examples. In case I want to create the superlatives of the words: tall -\\> tallest.\n```python\nfrom langchain import PromptTemplate, FewShotPromptTemplate\n\n\n# First, create the list of few shot examples.\nexamples = [\n    {\"word\": \"cool\", \"superlative\": \"coolest\"},\n    {\"word\": \"tall\", \"superlative\": \"tallest\"},\n]\n```\nNow we create the template as done before by including two variables, one for the base word and one for the superlative.\n```python\n# Next, we specify the template to format the examples we have provided.\n# We use the `PromptTemplate` class for this.\nexample_formatter_template = \"\"\"\nWord: {word}\nSuperlative: {superlative}\\n\n\"\"\"\nexample_prompt = PromptTemplate(\n    input_variables=[\"word\", \"superlative\"],\n    template=example_formatter_template,\n)\n```\nNow we combine everything with the FewShotPromptTemplate class that takes as input the examples, the template, the prefix which is usually some instruction we want to give to the template, and a suffix which is the form of the template output.\n```python\n# Finally, we create the `FewShotPromptTemplate` object.\nfew_shot_prompt = FewShotPromptTemplate(\n    examples=examples,\n    example_prompt=example_prompt,\n\n    prefix=\"Give the superlative of every input\",\n\n    suffix=\"Word: {input}\\nSuperlative:\",\n\n    input_variables=[\"input\"],\n    example_separator=\"\\n\\n\",\n)\n```\n```python\nprint(few_shot_prompt.format(input=\"big\"))\n```\n***My Output: **Give the superlative of every input Word: cool Superlative: coolest Word: tall Superlative: tallest Word: big Superlative:*\nNow you can feed the model and receive the actual output.\n```python\nllm(few_shot_prompt.format(input=\"large\"))\n```\n# **Indexes**\nThis module is the one that allows us to interact with external documents that we want to feed to the model. This module is based on the concept of Retriever. In fact, the most common thing we want to do with this module is to go and fetch the document that most answers our query. Thus an information retrieval system. Let’s see what the Retriever interface looks like to understand it better. ( For those who don’t already know, an interface is a class that cannot be instantiated, if you want to learn more you can read my articles on [design patterns](https://medium.com/towards-data-science/design-patterns-with-python-for-machine-learning-engineers-observer-23cde7ecb2ed)).\n```python\nfrom abc import ABC, abstractmethod\nfrom typing import List\nfrom langchain.schema import Document\n\nclass BaseRetriever(ABC):\n    @abstractmethod\n    def get_relevant_documents(self, query: str) -> List[Document]:\n        \"\"\"Get texts relevant for a query.\n\n        Args:\n            query: string to find relevant texts for\n\n        Returns:\n            List of relevant documents\n        \"\"\"\n```\nThe *get_relevant_documents* method is very simple, you just need to know how to read English to understand what it does. The string can be changed to your liking, so if you want to modify or implement a custom Retriever it is nothing too complicated.\nLet’s look at a practical example now. We want to create a question-answering application about a specific document. That is, the model needs to be able to answer questions of mine about a specific document.\nWe first need to install* Chroma* which allows us to work with *Vectorstores*, we’ll see later what it is for.\n```python\n!pip install chromadb >> null\n```\nLet’s import some classes we will need.\n```python\nfrom langchain.chains import RetrievalQA\nfrom langchain.llms import OpenAI\n```\nNow let’s download a txt document from the web.\n```python\n#download data\nimport requests\n\nurl = \"https://raw.githubusercontent.com/hwchase17/langchain/master/docs/modules/state_of_the_union.txt\"\nresponse = requests.get(url)\ndata = response.text\nwith open(\"state_of_the_union.txt\", \"w\") as text_file:\n    text_file.write(data)\n```\nLet’s load the document with the TextLoader.\n```python\nfrom langchain.document_loaders import TextLoader\nloader = TextLoader('state_of_the_union.txt', encoding='utf8')\n```\nThe *Retriever* always relies on what is called *Vectorstore* retriever. So we can instantiate one *Vectorstore retriever* and pass to it our text loader.\n```python\nindex = VectorstoreIndexCreator().from_loaders([loader])\n```\nNow that we finally have an index, we can ask questions about the data.\n```python\nquery = \"What did Ohio Senator Sherrod Brown say?\"\nindex.query(query)\n```\n***My Output:** Ohio Senator Sherrod Brown said, “It’s time to bury the label ‘Rust Belt.’”*\n# **Chains**\nChains allow us to create more complicated applications. **Often simply using an LLM is not enough**, we want to do more. For example, we want to first create a template and then give the compiled template as input to the LLM, this can be done simply with a chain. It’s easy for me to think of chains as the Pipeline that you use in scikit-learn, for example.\nThe LLMChain is a chain that does just that, takes an input, formats it within a template, and then passes it as input to a template.\nFirst, let’s create a simple template.\n```python\nfrom langchain.prompts import PromptTemplate\nfrom langchain.llms import OpenAI\n\nllm = OpenAI(temperature=0.9)\nprompt = PromptTemplate(\n    input_variables=[\"product\"],\n    template=\"What is a good name for a website that sells {product}?\",\n)\n```\nNow we create a chain by specifying the template and model to be used.\n```python\nfrom langchain.chains import LLMChain\nchain = LLMChain(llm=llm, prompt=prompt)\n\n#run the chain with the input needed for the prompt\nprint(chain.run(\"paints\"))\n```\nYou can also create a custom chain but I’ll write a more detailed article about it.\n# **Memory**\nEvery time we interact with a model, it will give us an answer that won’t depend on the context because it doesn’t remember past events. It’s kind of like every time we are talking to a new model. **In applications we usually want the model to have some kind of memory so that it learns to reply to us by improving as it goes along depending on what we said before** especially if we are developing chatbots. That’s what this module is for.\nMemory can be implemented in a variety of ways. For example, we can take the previous N messages, and feed them to the model as a single string or as a string sequence for instance. Now let’s look at the simplest type of memory we can implement called a buffer.\nWe can use a ChatMessageHistory class that allows us to easily save all the messages sent to the model.\n```python\nfrom langchain.memory import ChatMessageHistory\n```\n```python\nhistory = ChatMessageHistory()history.add_user_message(\"hello friend!\")history.add_ai_message(\"how are you?\")\n```\nNow we can retrieve the messages easily.\n```python\nhistory.messages\n```\nNow that we understand the concept, we can use a wrapper of the class we just used called ConversationBufferMemory that allows us to actually use the message history.\n```python\nfrom langchain.memory import ConversationBufferMemory\n```\n```python\nmemory = ConversationBufferMemory()\nmemory.chat_memory.add_user_message(\"hello friend!\")\nmemory.chat_memory.add_ai_message(\"how are you?\")\n```\n```python\nmemory.load_memory_variables({})\n```\nLet’s see finally how to use this feature in a chain for a chat.\nSo let’s start by having a conversation with the model.\n```python\nfrom langchain.llms import OpenAI\nfrom langchain.chains import ConversationChain\n\n\nllm = OpenAI(temperature=0)\nconversation = ConversationChain(\n    llm=llm,\n    verbose=True,\n    memory=ConversationBufferMemory()\n)\n```\n```python\nconversation.predict(input=\"Hello friend!\")\n```\n![](https://miro.medium.com/v2/resize:fit:700/1*Juxcji4qqwO49aQYRiq_Fg.png)\n```python\nconversation.predict(input=\"I would like to discuss about the universe\")\n```\n![](https://miro.medium.com/v2/resize:fit:700/1*Ec_xpQ9H9gtUnVjlDG0J-Q.png)\n```python\nconversation.predict(input=\"Whats your favourite planet?\")\n```\n![](https://miro.medium.com/v2/resize:fit:700/1*m1aYGseYRpUI0MtDPRBjDg.png)\nYou can retrieve the old messages by accessing the memory.\n```python\nconversation.memory\n```\nNow you can save your messages to load them again and feed the model when you want to start a conversation from a certain point.\n# **Agents**\nThe kind of chains we have seen, follow predetermined steps like a pipeline. But we often don’t know what steps the model has to take to answer a given question, because it also depends on what the user answers it from time to time.\nWe can have a model use various tools, to improve its answers. A common example is to first go and read some information on Wikipedia and then answer a particular question.\n```python\nfrom langchain.agents import load_tools\nfrom langchain.agents import initialize_agent\nfrom langchain.agents import AgentType\nfrom langchain.llms import OpenAI\n```\nWe are going to use two tools in particular SERPAPI which allows the model to do browser searches and take information, llm-math to improve its math skills.\nTo install SERPAPI you must register on the site and copy the API Token.\nThis is the website: [https://serpapi.com/](https://serpapi.com/)\nOnce done we install the library and set the token as an environment variable.\n```python\n!pip install google-search-results >> null\nos.environ['SERPAPI_API_KEY'] = \"your token here\"\n```\nNow we are ready to instantiate our agent with the tools it will need to use.\nWe, therefore, need 3 things:\n- LLM: the large language model we want to use\n- Tools: we want to use to improve the basic LLM\n- Agent: handles the interaction between the LLM and Tools\n```python\nllm = OpenAI(model_name=\"text-ada-001\")\n```\n```python\ntools = load_tools([\"serpapi\", \"llm-math\"], llm=llm)\n```\n```python\nagent = initialize_agent(tools, llm, agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION, verbose=True)\n```\nNow we can ask our model a question, relying on the fact that it is going to use the various tools available to it to answer us.\n```python\nagent.run(\"How old isJoe Biden? What is his current age raised to the 0.56 power?\")\n```\n# **Final Thoughts**\nIn this article, we introduced LangChain with its various modules.
