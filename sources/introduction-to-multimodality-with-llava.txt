But I believe that LLMs that work mainly on text is only the beginning of GenAI. At a certain point, everybody will want **physical AI**, where models can see, hear, feel, and reason in a more grounded, human way.\nSo let’s get started with multimodality. In this notebook, I introduce LLaVA, an architecture capable of interpreting both images and text to generate multimodal responses.\nIn this tutorial, we are going to use a lighter-weight component suitable to run the notebook on a free-tier environment such as Google Colab.\nThe components we are going to use are:\n1️⃣ **CLIP-ViT B/32** as the image encoder\n2️⃣ **TinyLlama-1.1B** as the language model\n3️⃣ A **2-layer MLP adapter** to bridge the two\n![](https://miro.medium.com/v2/resize:fit:1400/0*Xz2qrE1l_DTTRuxx.png)\nFrom the paper [Visual Instruction Tuning](https://www.google.com/url?q=https%3A%2F%2Farxiv.org%2Fpdf%2F2304.08485) (NeurIPS 2023)\n# **Setup**\nBefore we can dive into the code, let’s set up our environment.\nLet’s first install the datasets library.\n```plain text\n!pip install -U datasets\n```\nWe now need to import the required packages from Hugging Face and PyTorch. These imports provide pre-trained models and utilities for multimodal processing.\n```plain text\nimport json\nfrom pathlib import Path\n\nimport requests\nimport safetensors\nimport torch\nfrom datasets import load_dataset\nfrom huggingface_hub import hf_hub_download\nfrom PIL import Image\nfrom transformers import (\n    AutoConfig,\n    AutoTokenizer,\n    LlamaTokenizer,\n    LlavaConfig,\n    LlavaForConditionalGeneration,\n    LlavaProcessor,\n    Seq2SeqTrainer,\n    Seq2SeqTrainingArguments,\n)\nfrom transformers.models.clip.modeling_clip import CLIPVisionModel\nfrom transformers.models.clip.image_processing_clip import CLIPImageProcessor\n```\n# **Download pre-trained model components**\nOur LLaVA model will be composed of:\n- A pre-trained **CLIP** encoder [openai/clip-vit-base-patch32](https://huggingface.co/openai/clip-vit-base-patch32)\n![](https://miro.medium.com/v2/resize:fit:1400/1*r4HaCno3dXaGS5-0Qq-dIQ.png)\nImage source: [https://arxiv.org/pdf/2103.00020](https://arxiv.org/pdf/2103.00020)\n- A pre-trained **Tiny LLaMA** decoder [TinyLlama/TinyLlama-1.1B-Chat-v1.0](https://huggingface.co/TinyLlama/TinyLlama-1.1B-Chat-v1.0)\n- A 2-layer MLP projector between the two\nThe `hf_hub_download` is a hub we are exploring in order to retrieve pre-trained weights:\n```plain text\nvision_backbone_name = \"openai/clip-vit-base-patch32\"\ntext_backbone_name = \"TinyLlama/TinyLlama-1.1B-Chat-v1.0\"\n```\n```plain text\n_ = hf_hub_download(\n    vision_backbone_name, filename=\"pytorch_model.bin\", local_dir=\"/content\"\n)\n_ = hf_hub_download(\n    text_backbone_name, filename=\"model.safetensors\", local_dir=\"/content\"\n)\n```\n# **Model**\n## **Instantiate a new LLaVA model**\nLet’s now instantiate a new LlaVA model. As explained above, a LlaVA model is composed of two parts, a visual encoder and a textual decoder that we have just downloaded.\n```plain text\nvision_config = AutoConfig.from_pretrained(vision_backbone_name).vision_config\ntext_config = AutoConfig.from_pretrained(text_backbone_name)\n```\nWe specify the backbone models in the LlaVA config. We then instantiate the actual model with **`LlavaForConditionalGeneration(llava_config)`**.\n```plain text\nllava_config = LlavaConfig(vision_config=vision_config, text_config=text_config)\n```\n```plain text\nmodel = LlavaForConditionalGeneration(llava_config).cuda()\nmodel\n```\n## **Perform some surgical operations**\n![](https://miro.medium.com/v2/resize:fit:1400/1*3j686AgahwAMCvoeaug6jw.png)\n*Image source: *[*https://unsplash.com/photos/doctor-having-operation-E285pJbC4uE*](https://unsplash.com/photos/doctor-having-operation-E285pJbC4uE)\nPreviously, we said we could construct an LLaVA model by starting from a pre-trained image encoder and a pre-trained LLM. Let’s do just that!\nThe original LLaVA model is initialised from a **CLIP-ViT L/14** and a **Vicuna v1.5 7B**. To make things more manageable with the resources provided by the free plan of Google Colab, we’ll use a **CLIP-ViT B/16** and a **TinyLlama 1.1B**.\n**The only component we’ll train is a 2-layer MLP adapter in between them.**\nIn order to use the CLIP and TinyLlama models, we need to load their pre-trained weights. But these weights can come in different formats like .safetensors or .bin. The load_weights function handles this for us. It checks the file type and calls the right loading function.\n```plain text\ndef load_weights(path_to_weights: str):\n    if path_to_weights.endswith(\".safetensors\"):\n        return load_safetensors_weights(path_to_weights)\n    elif path_to_weights.endswith(\".bin\"):\n        return load_bin_weights(path_to_weights)\n    else:\n        raise ValueError(f\"Unsupported weights file: {path_to_weights}\")\n\ndef load_bin_weights(path_to_weights: str):\n    return torch.load(path_to_weights, weights_only=True)\n\ndef load_safetensors_weights(path_to_weights: str):\n    return safetensors.torch.load_file(path_to_weights)\n```\n```plain text\nvision_backbone_state_dict = load_weights(\"/content/pytorch_model.bin\")\ntext_backbone_state_dict = load_weights(\"/content/model.safetensors\")\n```\n## **Inject the vision backbone’s weights into the model**\nThe next lines loads the weights into the vision part of the model. We set *strict=False* to be flexible since it allows us to skip any weights that don’t perfectly match the model’s expected structure.\n```plain text\nincompatible_keys = model.vision_tower.load_state_dict(\n    vision_backbone_state_dict, strict=False\n)\n\nassert len(incompatible_keys.missing_keys) == 0, (\n    f\"Missing keys in state dict: {incompatible_keys.missing_keys}\"\n)\n\nincompatible_keys.unexpected_keys\n```\n## **Inject the text backbone’s weights into the model**\nSame logic as before, but also for the text model.\n```plain text\nincompatible_keys = model.language_model.load_state_dict(\n    text_backbone_state_dict, strict=True\n)\n```\n## **Freeze the pre-trained components**\nWe want now to freeze the backbone visual and text models, because we don’t want to update their weights while training.\nWe will only train the small adapter (the MLP that connects vision and language), which is much lighter and faster to train.\n```plain text\n_ = model.vision_tower.requires_grad_(False)\n_ = model.language_model.requires_grad_(False)\n```\n```plain text\n# Then we define a helper function to count model parameters\n\ndef count_parameters(model, trainable_only=False):\n    return sum(\n        p.numel()\n        for p in model.parameters()\n        if not trainable_only or p.requires_grad\n    )\n\nprint(f\"Total parameters: {count_parameters(model)}\")\nprint(f\"Trainable parameters: {count_parameters(model, trainable_only=True)}\")\n```\n## **Processor**\nBefore feeding some text into our model, we need to convert words into numbers. This is what the tokenizer is needed for.\n```plain text\ntokenizer = LlamaTokenizer.from_pretrained(\n    text_backbone_name, additional_special_tokens=[\"<image>\", \"<pad>\"]\n)\ntokenizer.pad_token_id = 32001\n```\nBelow is the format we’ll use to chat with our LLaVA model.\nThe first part is the so-called *system prompt*, which contains general guidelines for how the model should respond to the user.\nThe second part is a Jinja template (basically code) that determines how the conversation is rendered based on some structured input (see example below).\n```plain text\nLLAVA_CHAT_TEMPLATE = (\n    \"A chat between a curious user and an artificial intelligence assistant. The assistant gives helpful, detailed, and polite answers to the user's questions. \"\n    \"{% for message in messages %}{% if message['role'] == 'user' %}USER: {% else %}ASSISTANT: {% endif %}{% for item in message['content'] %}{% if item['type'] == 'text' %}{{ item['text'] }}{% elif item['type'] == 'image' %}<image>{% endif %}{% endfor %}{% if message['role'] == 'user' %} {% else %}{{eos_token}}{% endif %}{% endfor %}\"\n)\ntokenizer.chat_template = LLAVA_CHAT_TEMPLATE\n```\n```plain text\nsample_messages = [\n    {\n        \"content\": [\n            {\n                \"index\": 0,\n                \"text\": None,\n                \"type\": \"image\"\n            },\n            {\n                \"index\": None,\n                \"text\": \"\\nWhat potential activities might be popular at this location?\",\n                \"type\": \"text\"\n            }\n        ],\n        \"role\": \"user\"\n    },\n    {\n        \"content\": [\n            {\n                \"index\": None,\n                \"text\": (\n                    \"At this location, with a sandy path leading to the ocean where multiple boats, including \"\n                    \"sailboats, are moored, popular activities might include boating, sailing, swimming, and \"\n                    \"beachcombing. Additionally, the sandy path and shoreline provide an ideal setting for leisurely \"\n                    \"strolls and picnics, while the ocean view offers a serene environment for relaxation and \"\n                    \"photography. Depending on the specific area and available facilities, other water sports such as \"\n                    \"kayaking, paddleboarding, and snorkeling could also be prevalent.\"\n                ),\n                \"type\": \"text\"\n            }\n        ],\n        \"role\": \"assistant\"\n    }\n]\n```\nLet’s apply the chat template to our samples.\n```plain text\ntokenizer.apply_chat_template(\n    sample_messages, tokenize=False, add_generation_prompt=False\n)\n```\nAt this point we’ve set up our tokenizer and downloaded the vision model. We bring them together into one unified **processor**.\n```plain text\nprocessor = LlavaProcessor(\n    image_processor=CLIPImageProcessor.from_pretrained(vision_backbone_name),\n    tokenizer=tokenizer,\n    patch_size=model.config.vision_config.patch_size,\n)\nprocessor.chat_template = LLAVA_CHAT_TEMPLATE\n```\nSince we added special tokens like `<image>` and `<pad>` to our tokenizer earlier, the model needs to **adjust its vocabulary** to understand them too\n```plain text\nmodel.resize_token_embeddings(len(tokenizer), pad_to_multiple_of=8)\n```\n# **Dataset**\nLet’s download the dataset we are going to use from Hugging Face.\nThe dataset containing samples of image-text couples is publicly available and can be found [here](https://huggingface.co/datasets/HuggingFaceH4/llava-instruct-mix-vsft).\n```plain text\ntrain_dataset = load_dataset(\n    \"HuggingFaceH4/llava-instruct-mix-vsft\", split=\"train\", streaming=True\n)\n```\nWhat do our training examples look like?\n```plain text\nnext(iter(train_dataset))\n```\nHow do we build a batch of examples?\nThe following function takes raw image-text examples and turns them into model-ready inputs. It formats the messages using the chat template, processes both the text and image with the `LlavaProcessor` we defined previously, and creates proper training labels while ignoring padding.\n```plain text\ndef get_data_collator(processor, ignore_index):\n    def collate_examples(examples):\n        # Extract texts and images from the raw examples\n        texts = []\n        images = []\n        for example in examples:\n            messages = example[\"messages\"]\n            text = processor.tokenizer.apply_chat_template(\n                messages, tokenize=False, add_generation_prompt=False\n            )\n            texts.append(text)\n            images.append(example[\"images\"][0])\n\n        # Process the inputs (tokenize text and transform images)\n        batch = processor(texts, images, return_tensors=\"pt\", padding=True)\n\n        # Create labels\n        labels = batch[\"input_ids\"].clone()\n        if processor.tokenizer.pad_token_id is not None:\n            labels[labels == processor.tokenizer.pad_token_id] = ignore_index\n        batch[\"labels\"] = labels\n\n        return batch\n\n    return collate_examples\n\n#NOTE: this does a bit more than a collate function should...\n```\n# **Training**\nLet’s finally define the training arguments, including batch size, learning rate, total steps, and use mixed precision (fp16) for speed. We also avoid saving checkpoints to keep things light. Then we wrap everything into a `Seq2SeqTrainer`passing in the model, dataset, and our custom collator for image-text inputs.\n```plain text\nargs = Seq2SeqTrainingArguments(\n    output_dir=\"/content/training_output\",\n    per_device_train_batch_size=2,\n    gradient_accumulation_steps=4,\n    learning_rate=2e-4,\n    max_steps=350,\n    lr_scheduler_type=\"cosine_with_min_lr\",\n    lr_scheduler_kwargs={\"min_lr\": 2e-5},\n    warmup_ratio=0.05,\n    logging_strategy=\"steps\",\n    logging_steps=5,\n    fp16=True,\n    remove_unused_columns=False,  # Important!\n    optim=\"adamw_torch\",\n    report_to=\"none\",\n    save_strategy=\"no\",  # let's not save the checkpoint to disk, otherwise it'll take 5 mins\n)\n\ntrainer = Seq2SeqTrainer(\n    model=model,\n    args=args,\n    data_collator=get_data_collator(\n        processor, ignore_index=model.config.ignore_index,\n    ),\n    train_dataset=train_dataset,\n)\n```\n```plain text\ntrainer.train()\n```\n# **Inference**\nTo be noted that to make sure inference works as expected you should use heavier models, and train for longer time.\nWe’ll use this image for inference:\n![](https://miro.medium.com/v2/resize:fit:1400/1*qVRakjnaGB8xUSzFqKPzQw.png)\n*Image source: *[*https://it.wikipedia.org/wiki/Gioconda#/media/File:Mona_Lisa,_by_Leonardo_da_Vinci,_from_C2RMF_retouched.jpg*](https://it.wikipedia.org/wiki/Gioconda#/media/File:Mona_Lisa,_by_Leonardo_da_Vinci,_from_C2RMF_retouched.jpg)\n```plain text\nconversation = [\n    {\n        \"content\": [\n            {\n                \"type\": \"image\"\n            },\n            {\n                \"text\": \"\\nWhat is represented in the image?\",\n                \"type\": \"text\"\n            }\n        ],\n        \"role\": \"user\"\n    }\n]\n```\nIn this cell block as an example, we load an image from a URL and format a conversation using the chat template. The processor turns both into tensors. Then we move the input to the model’s device and generate a response, letting the model describe the image based on the user’s prompt.\n```plain text\nimage_url = \"https://llava-vl.github.io/static/images/monalisa.jpg\"\n\ninputs_for_generation = processor(\n    images=Image.open(requests.get(image_url, stream=True).raw),\n    text=processor.apply_chat_template(conversation, add_generation_prompt=True),\n    return_tensors=\"pt\",\n)\n\ninputs_for_generation = inputs_for_generation.to(device=model.device)\noutput = trainer.model.generate(\n    **inputs_for_generation, max_new_tokens=200, do_sample=False\n)\n```\n```plain text\nprint(processor.decode(output[0], skip_special_tokens=True))\n```\n# **Extensions and improvements**\n- Use a larger image encoder (e.g. **CLIP-ViT Large**) and LLM (e.g. **Llama 3.1 8B**)\n- Train for longer. It takes some time for the model to figure out how to follow instructions in the presence of image features\n- Follow the multi-stage training procedure adopted by the original LLaVA\n- **Stage 1: Pre-training for Feature Alignment** –\\> train the model on single-turn instruction data, where it is asked to briefly describe the picture. Image encoder and LLM are frozen\n- **Stage 2: Fine-tuning End-to-End** –\\> train the model on multi-turn instruction data.
