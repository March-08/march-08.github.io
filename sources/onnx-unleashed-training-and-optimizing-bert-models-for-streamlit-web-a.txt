**Introduction**\nIn this article, I want to accomplish something very simple: build a web app that recognizes an emotion given a sentence. In doing this though we will see how to train a transformer-based model, convert it to [ONNX](https://onnx.ai/) format, quantize it, and run it from the frontend using [Streamlit](https://streamlit.io/).\nYou can tun the following scripts using [Deepnote](https://deepnote.com/): a cloud-based notebook that’s great for collaborative data science projects, and good for prototyping.\nOptimizing the model with techniques such as quantization may be a good idea if we can maintain good performance, as it will improve the response speed, and we can create a product with lower latency and ensure greater user satisfaction.\nWe use a **BERT-based model for emotion detection**: anger, fear, joy, love, sadness, and surprise.\nThis is a model released by Microsoft, which is a **distilled version** of BERT.\n- model: [https://huggingface.co/microsoft/xtremedistil-l6-h256-uncased](https://huggingface.co/microsoft/xtremedistil-l6-h256-uncased)\n- dataset: [https://huggingface.co/datasets/dair-ai/emotion](https://huggingface.co/datasets/dair-ai/emotion)\nWe will heavily use the hugging face APIs to train this model on this dataset.\n## **Model Training**\nLet’s start by installing the needed libraries. We are going to use a lot of the transformers and ONNX ones.\n```plain text\n!pip install transformers[torch]\n!pip install datasets  onnx onnxruntime\n!pip install accelerate -U\n```\nAll the imports we need:\n```plain text\nfrom datasets import load_dataset\nfrom transformers import AutoTokenizer\nimport torch\nfrom transformers import AutoModelForSequenceClassification\nimport numpy as np\nfrom datasets import load_metric\nfrom transformers import TrainingArguments\nfrom transformers import Trainer\nimport transformers\nimport transformers.convert_graph_to_onnx as onnx_convert\nfrom pathlib import Path\nimport onnxruntime as ort\nfrom onnxruntime.quantization import quantize_dynamic, QuantType\nimport numpy as np\nfrom google.colab import files\n```\nI chose a [BERT-based model distilled by Microsoft](https://huggingface.co/microsoft/xtremedistil-l6-h256-uncased) because it's lighter than its original version.\n```plain text\nmodel_name = 'microsoft/xtremedistil-l6-h256-uncased'\n```\nWe obviously need some data to fine-tune our model. We can use the [emotion dataset](https://huggingface.co/datasets/dair-ai/emotion) that can be found on the HuggingFace platform.\n![](https://miro.medium.com/v2/resize:fit:1400/1*iMZyyIrML892_QysBB5PjQ.png)\nDataset View (src: [https://huggingface.co/datasets/dair-ai/emotion](https://huggingface.co/datasets/dair-ai/emotion))\nThis **dataset has a total of 20,000 examples** split into train, validation and split\nAs usual, we need to load the dataset and tokenize the text in order to be fed to the model. Each model has its own associated tokenizer that we can retrieve by using the AutoTokenizer class.\n```plain text\ndef tokenize_function(examples):\n    return tokenizer(examples[\"text\"], padding=\"max_length\", truncation=True, max_length=128)\n\ndataset = load_dataset(\"emotion\")\ntokenizer = AutoTokenizer.from_pretrained(model_name)\ntokenized_datasets = dataset.map(tokenize_function, batched=True)\n```\nThe next step is to split the training from the testing data.\n```plain text\nfull_train_dataset = tokenized_datasets[\"train\"]\nfull_eval_dataset = tokenized_datasets[\"test\"]\n```\nIf we have a GPU, it would be smart to set it up as the default training device. Otherwise, let's just use the CPU.\n```plain text\ndevice = \"cuda:0\" if torch.cuda.is_available() else \"cpu\"\nprint(device)\n```\nNow, we can download the model and specify that we need it for a classification task with 6 classes. Then, let’s move the model on the device (GPU). To better understand how that works, you might be interested in reading the BERT paper.\nLink to the paper: [https://arxiv.org/abs/1810.04805](https://colab.research.google.com/corgiredirector?site=https%3A%2F%2Farxiv.org%2Fabs%2F1810.04805)\n```plain text\nmodel = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=6)\nmodel = model.to(device)\n```\nWe should be capable of understanding how our model is doing on this task, which means that we need a way to evaluate it. It’s a classification task, so the evaluation method is quite easy. In this case, we can just rely on the accuracy metric so we can check the difference between the predicted labels and the real ones.\n```plain text\nmetric = load_metric(\"accuracy\")\n\ndef compute_metrics(eval_pred):\n    logits, labels = eval_pred\n    predictions = np.argmax(logits, axis=-1)\n    return metric.compute(predictions=predictions, references=labels)\n```\nThe transformer library allows us to create a *TrainingArguments* object where we can specify the training hyperparameters like *batch_size* and *learning_rate*.\nThen, pass these arguments to the *Trainer* together with the model and the data, and you’re ready to go.\n```plain text\ntraining_args = TrainingArguments(\"test_trainer\",\n                                  per_device_train_batch_size=128,\n                                  num_train_epochs=24,\n                                  learning_rate=3e-05,\n                                  evaluation_strategy=\"epoch\")\ntrainer = Trainer(\n    model=model,\n    args=training_args,\n    train_dataset=full_train_dataset,\n    eval_dataset=full_eval_dataset,\n    compute_metrics=compute_metrics,\n)\n```\nTraining might take a while, especially if you’re not using a GPU…\n```plain text\ntrainer.train()\n```\nGreat! Our model is now trained on the emotional data. How good is that, though? Let’s evaluate it!\n```plain text\ntrainer.evaluate()\n```\nAccuracy is greater than 90%.\nNow we can put both the tokenizer and the model into a pipeline so that we are able to classify new sentences.\n```plain text\npipeline = transformers.pipeline(\"text-classification\",model=model,tokenizer=tokenizer)\npipeline(\"I am very mad!\")\n```\n## **ONNX Conversion and Quantization**\nONNX is an intermediary machine learning framework used to convert between different machine learning frameworks.\nLet’s move the model back to the CPU.\n```plain text\nmodel = model.to(\"cpu\")\n```\nWith a single function, we can convert our developed [PyTorch](https://pytorch.org/) pipeline into an ONNX model.\nThe `opset` parameter specifies the version of the ONNX operator set to be used during the conversion from PyTorch to ONNX. The ONNX operator set defines the set of operations and their semantics that the ONNX model can use.\n```plain text\nonnx_convert.convert_pytorch(pipeline, opset=11, output=Path(\"classifier.onnx\"), use_external_format=False)\n```\nWe’re not done yet. We can still **quantize our model**. This means less precision in the weights representation, which makes the model smaller. In this case, I am using INT8 quantization.\n```plain text\nquantize_dynamic(\"classifier.onnx\", \"classifier_int8.onnx\",\n                 weight_type=QuantType.QUInt8)\n```\nTo use the ONNX models, you need you create a session. So I am instantiating here a session for both unquantized and quantized models.\n```plain text\nsession = ort.InferenceSession(\"classifier.onnx\")\nsession_int8 = ort.InferenceSession(\"classifier_int8.onnx\")\n```\nLet’s define a function to run the prediction on a single sentence by tokenizing the input sentence and running inference with the ONNX session.\n```plain text\ndef predict_sentece(sentence:str):\n\n  # Tokenize and preprocess the input sentence\n  tokens = tokenizer(sentence, return_tensors='np', padding=True, truncation=True)\n\n  # Create the input_data dictionary\n  input_data = {\n      'input_ids': tokens['input_ids'],\n      'attention_mask': tokens['attention_mask'],\n      'token_type_ids': tokens['token_type_ids'],\n  }\n\n  # Run inference\n  output = session.run(None, input_data)\n\n  return np.argmax(output[0], axis=-1)\n\n\npredict_sentece(\"I Love Python\")\n```\nAn input feed always looks like the following. You can read more about it here: [https://huggingface.co/learn/nlp-course/chapter2/4?fw=pt](https://colab.research.google.com/corgiredirector?site=https%3A%2F%2Fhuggingface.co%2Flearn%2Fnlp-course%2Fchapter2%2F4%3Ffw%3Dpt)\nWe have this data structure already ready because the downloaded dataset had this particular format. We can then exploit it for evaluation.\n```plain text\ninput_feed = {\n    \"input_ids\": np.array(full_eval_dataset['input_ids']),\n    \"attention_mask\": np.array(full_eval_dataset['attention_mask']),\n    \"token_type_ids\": np.array(full_eval_dataset['token_type_ids'])\n}\n```\nLet’s run the models and get the output with the help of the* numpy argmax* function.\n```plain text\nout = session.run(input_feed=input_feed,output_names=['output_0'])[0]\nout_int8 = session_int8.run(input_feed=input_feed,output_names=['output_0'])[0]\n```\n```plain text\npredictions = np.argmax(out, axis=-1)\npredictions_int8 = np.argmax(out_int8, axis=-1)\n```\nLet’s check to be sure that the accuracy of the predicted outputs using the converted ONNX model is still high.\n```plain text\nmetric.compute(predictions=predictions, references=full_eval_dataset['label'])\n```\nSame thing with the quantized model. You will notice that our quantized model, is lighter for sure, but its accuracy decreases by 10 points.\n```plain text\nmetric.compute(predictions=predictions_int8, references=full_eval_dataset['label'])\n```\nIf you are using Colab, you can download the two ONNX models by using this function.\n```plain text\nfiles.download('classifier_int8.onnx')\nfiles.download('classifier.onnx')\n```\n## **Deploy on a Streamlit Web App**\nNow that we have downloaded the two model files, we can use one to develop our sentiment classification app.\nI am not a good front-end developer, so when I need to create fast prototypes, I like using tools like Streamlit or [Gradio](https://www.gradio.app/).\nIn a few lines of Python code, we can set up this app easily.\nIn the following code we:\n- set up the session runtime ONNX\n- set up the tokenizer\n- create a dictionary to map the predicted classes to the real emotions\n- reuse the prediction function we developed earlier\n- set up a simple UI where the user can insert a prompt and run the model with a button.\n```plain text\nimport streamlit as st\nfrom transformers import pipeline\nimport onnxruntime as ort\nimport numpy as np\nfrom transformers import AutoTokenizer\n\nsession = ort.InferenceSession(\"classifier.onnx\")\nmodel_name = \"microsoft/xtremedistil-l6-h256-uncased\"\ntokenizer = AutoTokenizer.from_pretrained(model_name)\n\n\ndecode_sentiment = {\n    0: \"sadness\",\n    1: \"joy\",\n    2: \"love\",\n    3: \"anger\",\n    4: \"fear\",\n    5: \"surprise\",\n}\n\n\ndef predict_sentece(sentence: str):\n    # Tokenize and preprocess the input sentence\n    tokens = tokenizer(sentence, return_tensors=\"np\", padding=True, truncation=True)\n\n    # Create the input_data dictionary\n    input_data = {\n        \"input_ids\": tokens[\"input_ids\"],\n        \"attention_mask\": tokens[\"attention_mask\"],\n        \"token_type_ids\": tokens[\"token_type_ids\"],\n    }\n\n    # Run inference\n    output = session.run(None, input_data)\n\n    return np.argmax(output[0], axis=-1)\n\n\n\n\n# Streamlit app\ndef main():\n    st.title(\"Sentiment Classification App with BERT\")\n\n    # User input\n    user_input = st.text_area(\"Enter your text here:\")\n\n    # Make prediction when the user clicks the button\n    if st.button(\"Predict Sentiment\"):\n        if user_input:\n            # Perform sentiment prediction\n            sentiment = predict_sentece(user_input)[0]\n\n            # Display the result\n            st.success(f\"Sentiment: {decode_sentiment[sentiment]}\")\n        else:\n            st.warning(\"Please enter a text for sentiment prediction.\")\n\n\nif __name__ == \"__main__\":\n    main()\n```\nThe result should look something like the following.\n![](https://miro.medium.com/v2/resize:fit:1200/1*mkIYWsjTLUeCI0A7tsDQEA.gif)\nStreamlit Web App (Image by Author)\n# **Conclusions**\nIn this article, where we implemented a small full-stack app, we saw how to train a model, how to optimize it using quantization, and how to create a simple frontend prototype using Streamlit.\nOne interesting thing to know about ONNX is that it is a format that can be converted to many other frameworks.
