**Introduction**\nWhen working on data science projects, one fundamental pipeline to set up is the one regarding data collection. Real-world Machine Learning mainly differs from Kaggle-like problems because data is not static. We need to scrape websites, gather data from APIs, and so on. This way of collecting data might look chaotic, and it is! That’s why we need to structure our code following best practices to bring some sort of order to all this mess.\n# **The ORM software pattern**\nOnce you identified the sources from which you want to gather your data, you need to collect them in a structured way to store those in your database. For example, you might decide that in order to train your LLM what you need are data sources which contain 3 fields: *author, content, *and* link.*\nWhat you could do is to download the data, and then write SQL queries to store and retrieve data from your database. More commonly you might want to implement all the queries to perform **CRUD** operations. CRUD stands for create, read, update, and delete. These are the four basic functions of persistent storage.\nNow, what you can do is encapsulate the complexity of writing SQL queries into an ORM (**object-relational mapping**) class that handles all the database operations. The ORM will be capable of interacting with SQL databases such as MySQL or PostgreSQL.\n## **Let’ code**\nTo start with, we need to import the necessary libraries and define a base class for all the ORM models.\n```plain text\nfrom sqlalchemy import create_engine, Column, Integer, String\nfrom sqlalchemy.ext.declarative import declarative_base\nfrom sqlalchemy.orm import sessionmaker\n\n# Define the base class for the ORM models\nBase = declarative_base()\n```\nWe are ready now to define a model. A model represents a table in our database. Here I am interested in building a *document *class to store my data. We need a user ID which will be our primary key for this model. If you are familiar with SQL, you should be familiar with this. In the model class, we also need to specify the table name to which the Python class will be associated. Let’s also rewrite the magic method __repr__ to be sure our objects will be represented and printed in a good way.\n```plain text\n# Define the Document model\nclass Document(Base):\n    __tablename__ = 'documents'\n\n    id = Column(Integer, primary_key=True, autoincrement=True)\n    author = Column(String, nullable=False)\n    content = Column(String, nullable=False)\n    link = Column(String, nullable=False)\n\n    def __repr__(self):\n        return f\"<Document(id={self.id}, author='{self.author}', content='{self.content}', link='{self.link}')>\"\n```\nLet’s define a SQLite database\n```plain text\nengine = create_engine('sqlite:///documents.db', echo=True)\nBase.metadata.create_all(engine)\n```\nIt is also important to initialize a session with which we will actually be able to perform operations over our database.\n```plain text\n# Create a session factory\nSession = sessionmaker(bind=engine)\nsession = Session()\n```\nWe can add data to our database using Python!\n```plain text\n# Add some documents to the database\ndef seed_data():\n    doc1 = Document(author='John Doe', content='This is a sample document about SQLAlchemy.', link='http://example.com/doc1')\n    doc2 = Document(author='Jane Smith', content='Understanding relationships in SQLAlchemy.', link='http://example.com/doc2')\n    doc3 = Document(author='Alice Johnson', content='An introduction to Python ORM.', link='http://example.com/doc3')\n\n    session.add_all([doc1, doc2, doc3])\n    session.commit()\n```\nSame for querying the DB. We can also filter based on some attributes like the author's name.\n```plain text\n# Query the database\ndef query_data(author:str = None):\n\n    if not author:\n      print(\"\\nDocuments:\")\n      for document in session.query(Document).all():\n          print(document)\n\n    print(f\"\\nDocuments by {author}:\")\n    john_doe_docs = session.query(Document).filter(Document.author == author).all()\n    for document in john_doe_docs:\n        print(document)\n```\nCheck this [link](https://docs.sqlalchemy.org/en/20/orm/) to learn more about SQLAlchemy ORM.\n# **The ODM software pattern**\nThe ODM pattern which stands for **object document mapping** is extremely similar to the ORM pattern, but instead of working with relational databases, we work with NoSQL databases such as [MongoDB](https://www.mongodb.com/).\nODM maps object-oriented code to JSON-like documents and stores them in the database.\nThis means that the class we are going to build will shape how the JSON data is represented in the DB. For this reason, we will adopt Pydantic, which provides:\n- **validation**: automatically checks if the input data matches the expected types and constraints\n- **serialization**.: converts Python objects to and from formats like JSON or dictionaries.\n## **Let’ code**\nWe start with the imports as usual.\n```plain text\nfrom pydantic import BaseModel, Field\nfrom uuid import UUID, uuid4\nfrom typing import Any, Dict\nfrom pymongo import MongoClient\n```\nThe class we are going to build will inherit from Pydantic **BaseModel** to use Pydantics features.\nThe type UUID is used to specify that the id variable should be a universally unique identifier. The Field class is provided by Pydantic to add constraints to variables (for validation) or to add some metadata. *default_factory=uuid4 *means that when a document is created without an ID the method uuid4() is automatically called to create an ID. uuid4() generates a random number.\n```plain text\nclass Document(BaseModel):\n    id: UUID = Field(default_factory=uuid4)\n    author: str\n    content: str\n    link: str\n```\nNow I want to define a method to generate an instance of the class starting from a dictionary object.\nThis will be a class method. In this way, we have a reference to the class (cls) and can leverage this to use the [Pydantic](https://docs.pydantic.dev/latest/) feature to automatically convert a JSON-like object into a class instance.\nThe type returned is therefore a Document. We need to use the apexes ‘Document’ to return the type of the class we are implementing in the class itself.\n```plain text\n    @classmethod\n    def from_mongo(cls, data: Dict[str, Any]) -> 'Document':\n        \"\"\"\n        Convert a MongoDB document to a Document instance.\n        This ensures the MongoDB '_id' field is mapped to 'id'.\n        \"\"\"\n        if '_id' in data:\n            data['id'] = data.pop('_id')\n        return cls(**data)\n```\nIn a similar way we implement the to_mongo() method.\n```plain text\n\n    def to_mongo(self) -> Dict[str, Any]:\n        \"\"\"\n        Convert a Document instance to a MongoDB document.\n        This ensures the 'id' field is mapped to '_id'.\n        \"\"\"\n        data = self.dict()\n        data['_id'] = data.pop('id')\n        return data\n```\nThe entire class then would be the following:\n```plain text\nclass Document(BaseModel):\n    id: UUID = Field(default_factory=uuid4)\n    author: str\n    content: str\n    link: str\n\n    @classmethod\n    def from_mongo(cls, data: Dict[str, Any]) -> 'Document':\n        \"\"\"\n        Convert a MongoDB document to a Document instance.\n        This ensures the MongoDB '_id' field is mapped to 'id'.\n        \"\"\"\n        if '_id' in data:\n            data['id'] = data.pop('_id')\n        return cls(**data)\n\n    def to_mongo(self) -> Dict[str, Any]:\n        \"\"\"\n        Convert a Document instance to a MongoDB document.\n        This ensures the 'id' field is mapped to '_id'.\n        \"\"\"\n        data = self.dict()\n        data['_id'] = data.pop('id')\n        return data\n```\nWe can instantiate a MongoDB connection and use these methods to insert and fetch data easily to and from Mongo.\n```plain text\n# Example usage with MongoDB\ndef main():\n    # Connect to the local MongoDB instance\n    client = MongoClient(\"mongodb://localhost:27017/\")\n    db = client.test_database\n    collection = db.documents\n\n    # Create a new document instance\n    new_document = Document(author=\"Jane Doe\", content=\"Learning MongoDB with Pydantic.\", link=\"http://example.com/doc2\")\n\n    # Insert the document into MongoDB\n    inserted_id = collection.insert_one(new_document.to_mongo()).inserted_id\n    print(f\"Inserted document with _id: {inserted_id}\")\n\n    # Fetch the document back from MongoDB\n    mongo_document = collection.find_one({\"_id\": inserted_id})\n\n    # Convert the MongoDB document to a Document instance\n    document_instance = Document.from_mongo(mongo_document)\n    print(\"Fetched and converted to Document instance:\", document_instance)\n\n    # Clean up (delete the inserted document)\n    collection.delete_one({\"_id\": inserted_id})\n    print(\"Deleted the inserted document.\")\n\n# Run the example\nmain()\n```\nDone!\nThis is a very simple example. You might consider implementing some other methods to streamline the interaction with Mongo. For example, some methods you could implement are:\n- save(): to allow an instance of the model to be inserted into the database\n- get_or_create(): method which attempts to find a document in a database, if not existing it is created.\n- bulk_insert(): to insert multiple documents in the database\n- find(): search in the database using advanced filters\n- bulk_find(): find multiple documents\n- get_collection_name(): determines the name of the MongoDB collection\n# **Final Thoughts**\nIn this article, I described how to implement the ORM and ODM paradigms to streamline the work with relational and document-based databases.
