**Introduction**\nI often see data scientists getting interested in the development of LLMs in terms of model architecture, training techniques or data collection. However, I have noticed that many times, outside the theoretical aspect, in many people have problems in serving these models in a way that they can actually be used by users.\nIn this brief tutorial, I thought I would show in a very simple way how you can serve an LLM, specifically llama-3, using [BentoML](https://www.bentoml.com/).\n> BentoML is an end-to-end solution for machine learning model serving. It facilitates Data Science teams to develop production-ready model serving endpoints, with DevOps best practices and performance optimization at every stage.\n## **We need GPU**\nAs you know in Deep Learning having the right hardware available is critical. Especially for very large models like LLMs, this becomes even more important. Unfortunately, I don’t have any GPU 😔\nThat’s why I rely on external providers, so I rent one of their machines and work there. I chose for this article to work on [Runpod](https://www.runpod.io/) because I know their services and I think it is an affordable price to follow this tutorial. But if you have GPUs available or want to use any other provider, feel free to skip this part.\nFirst, make sure you have a Runpod account. Next we need to create a cryptographic key pair with which we can authenticate to Runpod via [SSH connection](https://www.geeksforgeeks.org/ssh-command-in-linux-with-examples/).\nI have a MacOS, so I used the following tutorial to generate the keys. But for Linux and Windows, the procedure should not be much different.\n[**Manually generating your SSH key in macOSYou generate an SSH key through macOS by using the Terminal application. Once you upload a valid public SSH key, the…**](https://docs.tritondatacenter.com/public-cloud/getting-started/ssh-keys/generating-an-ssh-key-manually/manually-generating-your-ssh-key-in-mac-os-x?source=post_page-----4d580a7a007f---------------------------------------)[<br>docs.tritondatacenter.com](https://docs.tritondatacenter.com/public-cloud/getting-started/ssh-keys/generating-an-ssh-key-manually/manually-generating-your-ssh-key-in-mac-os-x?source=post_page-----4d580a7a007f---------------------------------------)\nYou should now have a public and a private key. Be sure not to share your private key anywhere! In the settings, Runpod will ask you to enter the public key so that you can authenticate, so go ahead and copy it into the appropriate slot as shown.\n![](https://miro.medium.com/v2/resize:fit:1400/1*vc_pNjenW2ylIPui7ptajA.png)\nNow we are ready to create a pod, that is, a virtual machine that we can use to code. Click on the “*+Deploy*” button.\n![](https://miro.medium.com/v2/resize:fit:1400/1*AqgrZS9c292bg6OfC_89AQ.png)\nRunpod will ask you to specify which type of GPU you want to use. The price changes depending on the performance of the GPU you need. Ours is just a tutorial so we do not have extreme needs in terms of latency or throughput. In this case, I chose an RTX 4090.\n![](https://miro.medium.com/v2/resize:fit:1400/1*mi9-aKXLTNs2y_gRVk7s5w.png)\nYou can further modify the model and increase the disk size to 40GB.\n![](https://miro.medium.com/v2/resize:fit:1400/1*TVtt3KO8RYLKBHDP5SXeuA.png)\nIf you click on the “*connect*” button, Runpod will show you the commands you can use from bash to connect to the machine made available for you remotely.\n![](https://miro.medium.com/v2/resize:fit:1400/1*e6IblNTU4u1wAPXuZw5StQ.png)\nBefore we start working, however, there is one more thing we need to do. The command shown above specifies an IP address and a port.\nNow you need to go into the terminal and access the .ssh folder where you keep your SSH keys.\n![](https://miro.medium.com/v2/resize:fit:1400/1*ILAlvRIqabM-CPte8e5m4g.png)\nEdit the “*config*” file using the “*vim config*” command.\nAdd an entry to your file as in my case. I named the host “*bentoml*”, and added the IP, the port and also the path where my private key is located, this way when trying to connect to this host, the pc will automatically know where to find the key to connect.\n![](https://miro.medium.com/v2/resize:fit:1400/1*ZQKJ6qBUiADjX9Oae2m6pQ.png)\nIt would be nice though to **connect from VSCode** instead of using the CLI right? Just follow a few simple steps. Open VSCode and click on the blue arrows at the bottom left as shown in the image. In the drop-down menu now click on “*connect to host*”.\n![](https://miro.medium.com/v2/resize:fit:1400/1*f3m5M5Hr3gTt_8wBTdoxjQ.png)\nNow VSCode will know which hosts are available because we entered them first in the configuration file, in fact, it will recognize *bentoml* as the host. Click on it.\n![](https://miro.medium.com/v2/resize:fit:1400/1*3sFkVRs6jd6UH8C830oIsQ.png)\nYou are now inside the Runpod virtual machine! Open the */workspace* folder and you can start working.\n![](https://miro.medium.com/v2/resize:fit:1400/1*fD1OpOM5nQzk6j5p3oOzOQ.png)\n## **Serve with BentoML**\nSetting up the development environment with Runpod was probably the most complex part of this tutorial because BentoML makes serving llama-3 really easy.\nFirst of all, with the CLI we can clone the repository developed by the BentoML team.\n```plain text\ngit clone https://github.com/bentoml/BentoVLLM.git\n```\nIn the repository, we will find several examples of different models.\n![](https://miro.medium.com/v2/resize:fit:1400/1*VE-2H_nL6Dh5qdC1AXK44A.png)\nIn this case, we will specifically use llama3–8b-instruct. So we go into that folder.\n```plain text\ncd BentoVLLM/\ncd llama3-8b-instruct/\n```\nWe need to install all the necessary requirements.\n```plain text\npip install -r requirements.txt && pip install -f -U \"pydantic>=2.0\"\n```\nThe actual code is found in the *service.py* file.\nHowever, it will suffice for us to invoke the following command to serve the model.\n```plain text\nbentoml serve .\n```\nWhen the model is served an IP address will open to you to see the API locally. If you add the path “*/docs*” to the IP address, you will find the swagger with all the methods available.\n![](https://miro.medium.com/v2/resize:fit:1400/1*7ZbKJ2BrKslwRAr7WUiFYQ.png)\nYou see that the main API is */generate* in which you can enter a prompt and a system prompt and wait for the output of the model.\n![](https://miro.medium.com/v2/resize:fit:1400/1*ccz9Wd_phK7WHH0JT85IpQ.png)\n![](https://miro.medium.com/v2/resize:fit:1400/1*ZE7R4tPTGsNSHO1tsqfcCQ.png)\nOf course, in addition to the swagger, you can use the API from code for example if you want to develop your own custom frontend!\n## **Conclusion**\nIn this article, we saw how to connect to a remote machine using an SSH connection. In this tutorial we used Runpod but all providers follow a similar procedure.\nConnecting via SSH directly from VSCode is very useful, so that we can write code and visualize files from our favourite IDE, and we saw in this article how to do this in simple steps by registering the host data on the configuration file.\nIronically the serving of llama-3 was the fastest part of this simple tutorial, as thanks to bento we only need to call a command to have the model running and usable via the swagger.\nFollow me on [Medium](https://medium.com/@marcellopoliti) for more in-depth tutorials on BentoML 😁\n💼 [Linkedin](https://www.linkedin.com/in/marcello-politi/) ️\\| 🐦 [X (Twitter)](https://twitter.com/_March08_) \\| [💻](https://emojiterra.com/laptop-computer/) [Website](https://marcello-politi.super.site/)\n<empty-block/>\nThis article has been published on [Towards Data Science](https://medium.com/data-science/llmops-serve-a-llama-3-model-with-bentoml-4d580a7a007f)\n</content>\n</page>"}
