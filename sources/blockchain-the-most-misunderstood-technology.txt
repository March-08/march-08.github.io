For most people, web3 is simply a place where scammers gather, but I believe that **blockchain is one of the most misunderstood technologies ever created**. I will begin with the basics in chronological order, **starting with Bitcoin**, then move on to the development of **Smart Contracts and Ethereum**, and finish with a look at the intersection of **AI and web3**. 🚀\n<empty-block/>\n**Table of Contents**\n<table_of_contents color=\"gray\"/>\n# Introduction\nBitcoin is a collection of technologies that form the basis for a **new digital money ecosystem**. You can use bitcoin in the same way you use traditional currencies, exchanging it with other people for goods or services. A user can **prove ownership** of a certain amount of bitcoin **through a private key**, which works like a password.  This key can also be used to sign a transaction when the user wants to send bitcoin to someone else.\nYou can send bitcoin to a wallet by specifying the wallet address, which works a bit like an email address. \nUser **keys are stored in** what is called a **wallet**, which is an application that allows people to interact with the bitcoin network.\nThe **bitcoin network is peer to peer (p2p)**, which means it is distributed and has no central authority. A user can sign and send transactions without needing permission from a bank, deciding freely how to use their own money.\n<empty-block/>\nHow are bitcoin created? \n![](https://prod-files-secure.s3.us-west-2.amazonaws.com/6ade33c7-273d-4339-af90-45fa8fe845e9/08c4db4c-b8e6-43d3-bd0a-f9be2eea4f46/gold.png?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Content-Sha256=UNSIGNED-PAYLOAD&X-Amz-Credential=ASIAZI2LB466SQQ5MXTO%2F20260708%2Fus-west-2%2Fs3%2Faws4_request&X-Amz-Date=20260708T155230Z&X-Amz-Expires=300&X-Amz-Security-Token=IQoJb3JpZ2luX2VjEMD%2F%2F%2F%2F%2F%2F%2F%2F%2F%2FwEaCXVzLXdlc3QtMiJHMEUCIHKptoL%2B%2FpxGScU2H1%2B4LLYwmYbX6ukJSbeuh%2FTEp4y%2BAiEA7duy6Ll1WehU0WwokHOIcKYRUWQm7VO1PGx2eskDf%2BYqiAQIiP%2F%2F%2F%2F%2F%2F%2F%2F%2F%2FARAAGgw2Mzc0MjMxODM4MDUiDPo2U09HKRcItdkZFCrcA4aPeKmNfFnVIOTPtsneXYZuwg04NdPjZhIsofgM6oSQOD3Ghi55qkQfKwMBZ4BAbl0Xxsmn7eEqsbfD7cs7IN2B9lELXIT5jMTdl82LYInc3F5G%2BV4IB8WMJ%2Bd5MG9O4tDrkVT13HjWEEbvobCRe64MbQncFLiPlTCpQjlxPBgiu0g1l0zMucrKdWFkMfV55LQEsRZV5HrE%2BeG9c3fG4UKYzgqJQ8Jsclc8nF9oK5GfYShWefbRcOfUk4EQ3EFMNCts1tHKxWibF1qf9nvGPIuOcuuxlPtWPGrwKB6L1OG3tR5bHF%2FWnfxkDYh1ON3N2l9F4xfUjX6AxfYAuIF7ddMKOQnXjZFgHFpx0pvRmhhqunFs4xNBXOYP%2FVbizmg1GJUM11mOGSdsVO8Rm%2F%2F%2B2kjhOpAsUzLMn0PREloWzhJExLpK9fGjPDIfqKqzNSLZZoloQVSSBbElqHsCXfBI%2FquvAmxM9xVKbv67JFyikSFlyJAH8bAjt8AvXdeWjJiROqDITkTLrJgbpsd0l1dUYLamc6ZNtnoi4VF2R%2BbhHk02KCBBMzV0MtHtQ%2FMXQ7M9rgGe6BD3VgeRK0YXJhDB3rxyrF46pCqzDhfWljVHesUXwLrWhxbtLrHG14YVMIbYudIGOqUBnNkB%2BWVvZdwbHSACVKV%2B%2BkRHqxzRFttfT1bRKxMqZNlM02nDDlWYwxLruO%2Bd2EdqT43WrxaiahKocMhK789Guw86FO2hSuvn%2FlfFYaITmVTLaL8ruUwkN6%2FWLsvUe7zgVcPKke81YWxGXXsX5RVZq8NMT%2FZCduaWm2arPmZOS2civj2cAK68PVTf8g18wAoadqqmiFy1c%2BznLzMn9cMIlo6oPJXa&X-Amz-Signature=5f395ba119f3ba5e799401b76b4a7a6da92f11d0d30c28e1f890ede117224964&X-Amz-SignedHeaders=host&x-amz-checksum-mode=ENABLED&x-id=GetObject)\nThrough a process called **mining**, some users apply their computational power, using computers or servers, to solve complex mathematical problems. When they solve a problem, they are rewarded with new bitcoin. A** miner** is the only type of user who **can validate transactions and record them permanently on the blockchain**. In this way, miners take over the role of a central bank, but in a distributed form.\nOn average, **every 10 minutes a miner validates the transactions** by solving the mathematical problem. What happens when hardware becomes more powerful? The problem adjusts and becomes harder, so that a miner still wins on average every ten minutes 🧭\n<empty-block/>\nThe bitcoin protocol defines a fixed maximum number of coins that will ever be created. That **limit** is close to **21 million** and is expected to be reached **by the year 2140**.\n<empty-block/>\nWhat is the price of Bitcoin today? 💰 Check:\n- [BitcoinAverage](https://bitcoinaverage.com/markets/)\n- [CoinCap.io \\| Reliable Cryptocurrency Prices and Market Capitalizations](https://www.coincap.io/)\n<empty-block/>\nWe said that the best way for a user to interact with the bitcoin network is through a wallet. These wallets can be grouped into three types:\n- **Full node client**\n\tThis client stores the entire history of transactions, meaning all transactions are replicated many times. That is why we can say transactions are set in stone. A full node can independently validate the entire blockchain and any transaction. However, storing a full node is computationally expensive.\n- **Lightweight client**\n\tAlso called SPV (simple payment verification). This client connects to a full node to access the bitcoin network and get transaction information. It stores the user wallet locally.\n- **Third party API client**\n\tThis type of client interacts with the network through APIs instead of connecting to the network directly.\n# Deep Dive into Bitcoin Mechanics\nIn this chapter you will understand how trust is achieved as an emergent property in the Bitcoin protocol. owner (address) of the bitcoin\n- Output: specifies the amount that the new owner will receive and its address\n**Output \\< Input**, since the difference is the **transaction fee** that the user need to pay to miners for their work of validating the transaction and storing in a block in the blockchain \n<empty-block/>\nWhen I receive some money, then I am allowed to spend them again to buy something right? chain.**\n<empty-block/>\n![](https://prod-files-secure.s3.us-west-2.amazonaws.com/6ade33c7-273d-4339-af90-45fa8fe845e9/e709e56e-4515-4f5c-9b63-7b57b1da70a9/image.png?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Content-Sha256=UNSIGNED-PAYLOAD&X-Amz-Credential=ASIAZI2LB466SQQ5MXTO%2F20260708%2Fus-west-2%2Fs3%2Faws4_request&X-Amz-Date=20260708T155230Z&X-Amz-Expires=300&X-Amz-Security-Token=IQoJb3JpZ2luX2VjEMD%2F%2F%2F%2F%2F%2F%2F%2F%2F%2FwEaCXVzLXdlc3QtMiJHMEUCIHKptoL%2B%2FpxGScU2H1%2B4LLYwmYbX6ukJSbeuh%2FTEp4y%2BAiEA7duy6Ll1WehU0WwokHOIcKYRUWQm7VO1PGx2eskDf%2BYqiAQIiP%2F%2F%2F%2F%2F%2F%2F%2F%2F%2FARAAGgw2Mzc0MjMxODM4MDUiDPo2U09HKRcItdkZFCrcA4aPeKmNfFnVIOTPtsneXYZuwg04NdPjZhIsofgM6oSQOD3Ghi55qkQfKwMBZ4BAbl0Xxsmn7eEqsbfD7cs7IN2B9lELXIT5jMTdl82LYInc3F5G%2BV4IB8WMJ%2Bd5MG9O4tDrkVT13HjWEEbvobCRe64MbQncFLiPlTCpQjlxPBgiu0g1l0zMucrKdWFkMfV55LQEsRZV5HrE%2BeG9c3fG4UKYzgqJQ8Jsclc8nF9oK5GfYShWefbRcOfUk4EQ3EFMNCts1tHKxWibF1qf9nvGPIuOcuuxlPtWPGrwKB6L1OG3tR5bHF%2FWnfxkDYh1ON3N2l9F4xfUjX6AxfYAuIF7ddMKOQnXjZFgHFpx0pvRmhhqunFs4xNBXOYP%2FVbizmg1GJUM11mOGSdsVO8Rm%2F%2F%2B2kjhOpAsUzLMn0PREloWzhJExLpK9fGjPDIfqKqzNSLZZoloQVSSBbElqHsCXfBI%2FquvAmxM9xVKbv67JFyikSFlyJAH8bAjt8AvXdeWjJiROqDITkTLrJgbpsd0l1dUYLamc6ZNtnoi4VF2R%2BbhHk02KCBBMzV0MtHtQ%2FMXQ7M9rgGe6BD3VgeRK0YXJhDB3rxyrF46pCqzDhfWljVHesUXwLrWhxbtLrHG14YVMIbYudIGOqUBnNkB%2BWVvZdwbHSACVKV%2B%2BkRHqxzRFttfT1bRKxMqZNlM02nDDlWYwxLruO%2Bd2EdqT43WrxaiahKocMhK789Guw86FO2hSuvn%2FlfFYaITmVTLaL8ruUwkN6%2FWLsvUe7zgVcPKke81YWxGXXsX5RVZq8NMT%2FZCduaWm2arPmZOS2civj2cAK68PVTf8g18wAoadqqmiFy1c%2BznLzMn9cMIlo6oPJXa&X-Amz-Signature=218c5d2b6c8612dcbd34b3719f1ff8d14d15a30992acaeeb38e858f1b9ecbeac&X-Amz-SignedHeaders=host&x-amz-checksum-mode=ENABLED&x-id=GetObject)\n<empty-block/>\n<columns>\n\t<column>\n\t\t### The mining process has 2 ojectives:\n- validating transactions\n- creating new bitcoin , as reward for the computational task that requires a lot of electricity\n<empty-block/>\nMiners try to find a solution to a complicated mathematical puzzle that resets every time (10 minutes) a solution is found by some miner.\nThe process to find the solution is the so called **Proof-of-Work (PoS).**\n<empty-block/>\nThe PoS basically consists in [hashing](https://en.wikipedia.org/wiki/Hash_function) (SHA 256) the **block header (**i.e the block ID) together with a random number, until the obtained number is smaller than a given number (by changing the given number, we change the difficulty of this challenge). \nThe first miner to find this solution can publish the block into the blockchain. \n<empty-block/>\nNew transactions are constantly flowing into the network, and they are added to a **temporary pool of unverified transactions maintained by each node.  **Miners get transactions in this pool, prioritising based on the transaction fee value,  and group them into blocks that they try to mine. \nI miner includes into the block also a special transaction with it’s own address which allows him to get the new bitcoins reward + the total amount of fees of all the transaction validated. \n<empty-block/>\nit is worth saying that each block contain in his own header a pointer to the previous block. Language\nBitcoin transaction script language is called ***Script —\\> ***simple language, limited scope, requires minimal processing. \nScript it is not a [Turing Complete](https://en.wikipedia.org/wiki/Turing_completeness) language (e.g no for loops).\n<empty-block/>\n- Locking script: is a spending condition put on the output of a transaction and it’s called `scriptPubKey` (or witness script or cryptographic puzzle)\n- Unlocking script: script that solves the locking puzzle. These are part of every tx input. In most cases they contain a digital signature produced by the user’s private key.  Called also `scriptSig` or `witness`\n<empty-block/>\nEvery bitcoin validating node will validate transactions by executing locking and unlocking scripts in sequence. \nThe Script language is havely based on a **stack**, meaning it can push and pop vars on the stack and do operations on them.\n<empty-block/>\n**Locking Script looks like this**\n```c\nOP_DUP OP_HASH160 7f9b1a7fb68d60c536c2fd8b8a8f3f3e8c9d2e3f OP_EQUALVERIFY OP_CHECKSIG\n```\n- `OP_DUP`: Duplicates the top stack item.\n- `OP_HASH160`: Hashes the duplicated public key.\n- `7f9b1a7fb68d60c536c2fd8b8a8f3f3e8c9d2e3f`: The expected public key hash.\n- `OP_EQUALVERIFY`: Checks if the hashes match; fails if not.\n- `OP_CHECKSIG`: Verifies the signature against the public key.\n<empty-block/>\n**Unlocking Script looks like this**\n```c\n<signature> <public key>\n```\n- `<signature>:` The ECDSA signature (e.g., 3044022047ac...) proving ownership.\n- `<public key>`: The full public key (e.g., 04x9...) that hashes to 7f9b1a7fb68d60c536c2fd8b8a8f3f3e8c9d2e3f.\n<empty-block/>\n**Validation step**\nCombine the scripts —\\> unlocking \\<\\> locking  \n<columns>\n\t<column>\n\t\tScript Execution (Step-by-Step, follow the image on the right) \n\t\t1. **Push** `<signature>`:\n\t\t\t- Stack = `[signature]`\n\t\t\t- The signature (e.g., `3044022047ac`...) is placed on the stack.\n\t\t2. **Push** `<public key>`:\n\t\t\t- Stack = `[signature, public key]`\n\t\t\t- The public key (e.g., `04x9`...) is added on top.\n\t\t3. **OP_DUP**:\n\t\t\t- Duplicates the top stack item (public key).\n\t\t\t- Stack = `[signature, public key, public key]`\n\t\t4. **OP_HASH160**:\n\t\t\t- Hashes the top public key (using SHA-256 followed by RIPEMD-160).\n\t\t\t- Stack = `[signature, public key, hash of public key]`\n\t\t5. digital signature used in bitcoin is the ***Elliptic Curve Digital Signature (ECDSA), ***and it is used by some script functions such as OP_CHECKSIG we saw above.\nIn bitcoin the signatures proves the ownership of a private key, so the authorization to spend the funds. \n<empty-block/>\nA digital signature is composed of two parts\n- An algorithm to create a signature using the private key from a given message (or transaction)\n- An algorithm to allow anyone to verity the signature given also the message and the public key\nIn math terms the signature is: \n$$\nSig = F_{sig}(F_{hash}(m),dA)\n\n\n$$\nwhere:\n- `dA` is the signing private key\n- `m `is the transaction (message)\n- `F_hash` is the hashing function\n- `F_sig` is the signing algorithm\n<empty-block/>\nSig at the end will be composed of two values R and S.\n$$\nSig = (R, S)\n$$\nMore in detail…\nThe signature algorithm generates a temporary private/public key pair which is based on a random number `k`. From `k` we generate the public key with the Formula `P=k*G` as we have seen already in a previous chapter.\nThe **R **value of the digital signature is the x coordinate of this new public key P.\nFrom there the algorithm calculates the S value such that:\n$$\nS = k^{-1}(Hash(m)+dA*R) \\mod(p)\n$$\nwhere:\n- `k` is the temporary private key\n- `R` is the x coordinate of the temporary public key P\n- `dA` is the signing private key\n- `m` is the transaction data (message)\n- `p` is the prime order of the elliptic curve\n<empty-block/>\nThe signature verification algorithm takes the message, the signer public key and the signature and returns TRUE is the signature is valid for the message and public key. \nThe verification is the inverse, it uses R, S,k to calculate P which is a value on the elliptic curve (the temporary public key used before).\n$$\nP = S^{-1} * Hash(m) * G + S^{-1} * R * Qa\n$$\nwhere:\n- `R` and `S` are the signatures value\n- `Qa` is the real public key (not the temporary one)\n- `m` is the transaction data that was signed\n- `G` is the elliptic curve generator point\n<empty-block/>\nIf the x coordinate of the calculated point P is equal to R then the verifier can conclude that the signature is valid. \n<empty-block/>\nIf you’re curious about the validation of the algorithm please refer to [Elliptic Curve Digital Signature Algorithm](https://en.wikipedia.org/wiki/Elliptic_Curve_Digital_Signature_Algorithm)\n<empty-block/>\n<empty-block/>\n# The Bitcoin Network\nBitcoin is structured as a [**peer-to-peer**](https://en.wikipedia.org/wiki/Peer-to-peer)** (P2P)** network, meaning there are no special nodes,  servers or hierarchy. Suppose we have four transactions: A,B,C and D. These are the leaves of our tree. \nThe transactions are not stored in the tree, but we hash their data and store the resulting hash: $`H_A,H_B,H_C`$ and $`H_D`$.\n$$\nH_A = SHA256(SHA256(A))\n$$\n<empty-block/>\nConsecutive pairs of leaf nodes are concatenated and then hashed together, to form their parent node.\n$$\nH_{AB} = SHA256(SHA256(H_A+H_B))\n$$\n \nThe process continue until we get only one node, known as the **markle root**.\nThe hash of the root, 32 bytes,  is stored in the block header and summarises all the data in all four transactions. \nNote that even with thousands of transactions, the final hash size will always be 32 bytes. We want to find a hash hexadecimal number that starts with 0. \n<empty-block/>\n```c\nHash(\"my name is Marcello Politi-1\") --> \"a591a6d40bf420404a011733cfb7b190d62c65bf0bcda32b57b277d9ad9f146e\"\nHash(\"my name is Marcello Politi-2\") --> \"7d1e3ed9e1e3b6e4a9d0b7a7f8c2e4b8a8b4c1e2d2f3a5b6c7d8e9f0a1b2c3d4\"\n...\n...\n...\nHash(\"my name is Marcello Politi-16\") --> \"0e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8\"\n```\nFound! Marcello’s transaction, and gives the car’s key to Marcello which immediately leaves.<br>\nCristiano, which is Marcello accomplice and controls 51% of the network hashing power, mines a new block at the same height of the block that contains Marcello’s transaction, creating then a fork.\nIn this new block Cristiano replaces the Marcello’s transaction with the Davide’s payment with a new payment that goes back to Marcello himself. \nThen Cristiano starts mining new blocks on top of this new forked block, making this secondary chain longer than the main one, which then actually becomes the main chain.\nNow Davide is missing is payment and doesn’t have the car no more. Hard and Soft Forks\nA hard fork is the scenario in which the participants of the network, split into following two chains.\nThis because in the new chain the consensus rules have changed completely and so the network is operating differently. The two chain in this case will evolve independently and won’t merge anymore.  \nNot all consensus rule changes cause hard fork, only those that are not forward-compatible. \nIf a new fork implements new rules, but old clients still see new mined block as valid, we call the this a soft-fork.\nSo transactions and blocks created under the new rules must be valid under the old rules too, but not vice versa. \n# Blockchain Applications\nBitcoin is a decentralised currency system, but the underline technology, the blockchain, is a tool that offers a trust service. that can be used to develop other applications for different use cases.
