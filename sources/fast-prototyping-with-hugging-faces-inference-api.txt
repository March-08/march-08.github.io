**Introduction**\nThe boom in Large Language Models has brought many people, not just data scientists, closer to the world of AI. Everyone is using these models in their applications, and **you certainly don’t have to be a Machine Learning** expert to do so, you just need to **invoke APIs served by pre-trained models** and get their output! So let’s see how to create and use APIs that are useful for doing real-time inference on real data created by experts and that can be integrated into more complex applications.\nLLMs are huge models that contain millions of parameters and have taken huge amounts of textual data (e.g., everything on the world wide web) and are able to process it and solve tasks. **One of the most curious things about these models is that they generate emergent capabilities**, on which they have never been trained. For example, they are able to solve small math or logic problems without having been specifically trained on them. Or they are able to interact with the user as in a chat room.\nMany realities are only recently approaching the world of AI and want to use Machine Learning capabilities to create products or offer new services. **Often to create a valuable AI-based product, the speed with which you can launch it into the market is critical.** So in this article, I want to talk about how to use Hugging Face for fast prototyping.\n## **Hugging Face**\n**Hugging Face** (HF) was established as a **leading platform for research and development in Natural Language Processing**. HF is based on an open and collaborative research approach, engaging developers and researchers from around the world. Users can share their projects, exchange ideas and collaborate to improve and develop new language models. This open and shared approach fosters growth and innovation in the field of artificial intelligence, accelerating progress and discovery.\nOne of the most widely used services that HF provides is the API, this allows you to provide a way for developers to interact with the AI models you have developed.\nThere are several models that solve different tasks whose APIs can be exposed. Let us look at some models and use cases:\n- **Classification Models**: Spam detection, Sentiment analysis, Topic labelling\n- **Question & Answer Models**: Online help centres, Virtual customer assistance\n- **Conversational Agents Models**: Interactive chat fiction, Voice assistant integration\n- **Image Caption Generation Models**: Stock photo optimization, Content creation acceleration\n- **Speech Synthesis Models**: IVR automation, Custom audiobooks/podcast narration\n- **Writing Generation Models**: Personalized greeting cards, Email subject line optimization.\n- **Named Entity Recognition Models**: News article tagging, CRM management, Social media monitoring\n- **Sentiment Analysis Models**: Opinion mining for surveys, Customer service chatbots, Marketing campaign evaluation\n- **Machine Translation Models**: Multilingual product descriptions, Foreign correspondence handling, Travel industry chatbots\n- **Chatbot Development Models**: Intelligent virtual agents, Sales lead generation\n# **Hugging Face Inference API**\nLet’s start by registering at the Hugging Face website.\n![](https://miro.medium.com/v2/resize:fit:700/1*SxRh3KGwBCvZ8GTOuy25kw.png)\nIn your account settings, you will see your API Token that you will need to use the template.\n![](https://miro.medium.com/v2/resize:fit:700/1*tFRh09ihSXSrL5m6mEORGg.png)\nTake a look at the token and save it.\nThen choose a model you want to use, depending on the problem you want to solve. For example, now I want to look for a Text Classification — Sentiment Analysis model.\n![](https://miro.medium.com/v2/resize:fit:700/1*mQ3I1kcVcuR3K8GEhiHNMQ.png)\nI now choose a model I am interested in from a list of models, there are so many of them!\n![](https://miro.medium.com/v2/resize:fit:700/1*K_YnLbh0xawGK47xv1yxIA.png)\nA **model is always accompanied by a model card**, which has sometimes is more or less detailed about the characteristics of the model and how it was trained etc..\n![](https://miro.medium.com/v2/resize:fit:600/1*zfZd3BcIwmBfitFE6CPulw.gif)\nIf you click on the deploy button in the upper right-hand corner, a drop-down menu will appear, where you can select the **Inference API** item.\n![](https://miro.medium.com/v2/resize:fit:700/1*rk1mRBAsJdYja1b2TsyOYA.png)\nInference API will open a window for you where it will explain step by step how to use the model in inference. These APIs are very useful because they allow you to create prototypes if you are working on AI-based projects very quickly.\n![](https://miro.medium.com/v2/resize:fit:700/1*Vq3vIq34n9lwWWWX3w1WpQ.png)\nSo let’s try now to follow the directions in this window and run the code.\n```plain text\nimport requests\n\nAPI_URL = \"https://api-inference.huggingface.co/models/cardiffnlp/twitter-roberta-base-sentiment\"\nheaders = {\"Authorization\": \"Bearer API_TOKEN\"}\n\ndef query(payload):\n response = requests.post(API_URL, headers=headers, json=payload)\n return response.json()\n\noutput = query({\n \"inputs\": \"Hey, dude! You're great!\",\n})\n\n#OUTPUT\n'''\n[[{'label': 'LABEL_0', 'score': 0.002866477705538273},\n  {'label': 'LABEL_1', 'score': 0.018881773576140404},\n  {'label': 'LABEL_2', 'score': 0.9782518148422241}]]\n'''\n```\nAs you can see by trying it, we get a result that is a score of whether the sentence is negative, neutral, or positive.\nLet’s look at another example with another model now. I would like to use a **Question Answering model** now, where we feed the model the context and a question, and the model will be able to answer the question by reading the context. First, we select the model.\n![](https://miro.medium.com/v2/resize:fit:700/1*ZQ05ifx6RU-erIrFoxvSKQ.png)\nWe now use the Inference API.\n```plain text\nimport requests\n\nAPI_URL = \"https://api-inference.huggingface.co/models/valhalla/longformer-base-4096-finetuned-squadv1\"\nheaders = {\"Authorization\": \"Bearer XXXXXXXXXXXXXXXXXX\"}\n\ndef query(payload):\n response = requests.post(API_URL, headers=headers, json=payload)\n return response.json()\n\noutput = query({\n \"inputs\": {\n  \"question\": \"What's my name?\",\n  \"context\": \"My name is Marcello and I am an AI specialist. I love pizza and I am Italian.\"\n },\n})\n\n# OUTPUT\n#{'score': 0.9990959763526917, 'start': 11, 'end': 19, 'answer': 'Marcello'}\n```\nLet us now try a **summarization model**. So as the name already suggests, the model will return the summary of a given text.\n```plain text\nimport requests\n\nAPI_URL = \"https://api-inference.huggingface.co/models/facebook/bart-large-cnn\"\nheaders = {\"Authorization\": \"Bearer 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\"}\n\ndef query(payload):\n response = requests.post(API_URL, headers=headers, json=payload)\n return response.json()\n\noutput = query({\n \"inputs\": \"The tower is 324 metres (1,063 ft) tall, about the same height as an 81-storey building, and the tallest structure in Paris. Its base is square, measuring 125 metres (410 ft) on each side. During its construction, the Eiffel Tower surpassed the Washington Monument to become the tallest man-made structure in the world, a title it held for 41 years until the Chrysler Building in New York City was finished in 1930. It was the first structure to reach a height of 300 metres. Due to the addition of a broadcasting aerial at the top of the tower in 1957, it is now taller than the Chrysler Building by 5.2 metres (17 ft). Excluding transmitters, the Eiffel Tower is the second tallest free-standing structure in France after the Millau Viaduct.\",\n})\n```\nOutput: *The tower is 324 metres (1,063 ft) tall, about the same height as an 81-storey building. Its base is square, measuring 125 metres (410 ft) on each side. During its construction, the Eiffel Tower surpassed the Washington Monument to become the tallest man-made structure in the world.*\n# **Final Thoughts**\nIn these examples, I used mainly NLP models, which therefore work on language.
