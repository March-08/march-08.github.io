**Introduction**\nI recently started working in the field of computer vision. And in these early days, I’m studying how the various **algorithms of object detection** work. Among the most well-known ones are **R-CNN**, **Fast R-CNN**, **Faster R-CNN **and of course **YOLO**.\nIn this article, I want to focus on the last mentioned algorithm. YOLO is the state of the art in object detection and there are endless use cases where YOLO can be used. Today, however, I don’t want to tell you about how YOLO works and its architecture, but** I want to show you simply how you can launch this algorithm and make your predictions**. Plus we’ll also see how it’s possible to** train it on a custom dataset** so you can adapt it to your data.\nIf you would also like to read an article of mine on the inner workings of YOLO follow me because I have proposed to write it in the next few days.\n## **Darknet**\nI don’t think there is any better way to describe Darknet than the definition you can find on their site at this [link](https://pjreddie.com/darknet/).\n> Darknet is an open source neural network framework written in C and CUDA. It is fast, easy to install, and supports CPU and GPU\n\tcomputation. You can find the source on GitHub or you can read more about what Darknet can do right [here](https://pjreddie.com/darknet/).\nSo all we will have to do is learn how to use this open-source project.\nYou can find the darknet code on [github](https://github.com/AlexeyAB/darknet). Take a look at it because we are going to use it to train YOLO on our custom dataset.\n## **Clone Darknet**\nThe code I’ll show you in this article below is meant to be run on Colab, since I didn’t have a GPU with me… of course you can also repeat this code on your notebook.\nWhat will change occasionally are the paths.\nSo first we go to clone the darknet GitHub repository. Colab allows us to write bash commands if we use the ***%%bash **command.*\n```plain text\n%%bash\ngit clone https://github.com/AlexeyAB/darknet\n```\nOnce you clone the repo you will see lots of files in your working directory, relax it looks more complicated than it is.\nNow we need to** reconfigure the makefile**. Don’t know what a makefile is? To simplify it is a file that makes it easy to compile your code.\nIf you have ever written code in C, you know that the practice is to write a file *file.c* after which you compile it using a command like *g++ etc…*\nThis command used to compile could be very long in large projects since it has to take into account dependencies and so much more.\nSo it would be really laborious every time to compile by going to rewrite *g++ etc…*\nThen what we do is create a makefile that already contains this written command inside, and all we would have to do is **launch the makefile to compile the code**.\nThe **makefile usually contains within itself configuration variables** that the user can set as desired.\nThat said what we want to do is set some variables that we find in the Darknet makefile.\nSo make sure you have a GPU available and run the following cell.\n```plain text\n%%bash\ncd darknet\nsed -i 's/OPENCV=0/OPENCV=1/' Makefile\n# In case you dont have a GPU, make sure to comment out the\n# below 3 lines\nsed -i 's/GPU=0/GPU=1/' Makefile\nsed -i 's/CUDNN=0/CUDNN=1/' Makefile\nsed -i 's/CUDNN_HALF=0/CUDNN_HALF=1/' Makefile\n```\nIn this cell command *sed -i* for example in the first line allows you to change the OPENCV variable from 0 to 1.\nThe setting we set in the previous cell allows us to launch YOLO on the GPU instead of the CPU.\nNow we are going to launch the makefile with the make command.\n```plain text\n%%bash\n#compile darkent source code\ncd darknet\n```\nNow we install a library that will serve among other things to draw bounding boxes around the objects detected by YOLO.\n```plain text\n%%capture\n!pip install -q torch_snippets\n```\n## **Download the dataset**\nI will use an object detection dataset containing images of trucks and buses. There are a lot of object detection datasets on Kaggle and you can download one from there.\nIf you don’t know how to download a Kaggle dataset directly from Colab you can go and read some of my [previous articles](https://towardsdatascience.com/transfer-learning-data-augmentation-for-robust-image-classification-643ca88b3175).\nSo I download and unzip the dataset.\n```plain text\n!wget - quiet link_to_dataset\n!tar -xf open-images-bus-trucks.tar.xz\n!rm open-images-bus-trucks.tar.xz\n```\nThe structure of the downloaded dataset is depicted in the following figure.\n![](https://miro.medium.com/v2/resize:fit:344/1*Wk8PbW8Zcq0sFo39G1_m9w.png)\nImage By Author\n## **Download YOLO**\nObviously, you will not have to do the YOLO training from scratch but will go and **download the weights **directly from the Internet.\nTo download the weights from **YOLO4** use the following command.\n```plain text\n!wget - quiet https://github.com/AlexeyAB/darknet/releases/download/darknet_yolo_v3_optimal/yolov4.weights\n```\nTo see if everything is working properly run the following command.\n```plain text\n%%bash\n#I had to use the flag -dont_show cause wasnt working. Try to run wiithout it\ncd darknet\n./darknet detector test cfg/coco.data cfg/yolov4.cfg ../yolov4.weights data/person.jpg -dont_show\n```\nIn this command, we have just run\n- we specify that we want the configuration of YOLO4: *cfg/yolov4.cfg*\n- We specify to use the weights we just downloaded: *../yolov4.weights*\n- We are going to do the prediction on the coco dataset that you have since you cloned the repo: *cfg/coco.data*\n- And we do the prediction of the following image: *data/person.jpg*\n## **Prepare your dataset**\nYOLO expects to find certain files and folders set up correctly in order to do the training on your custom dataset.\nFirst, you will need to open the file in the *darknet/data/obj.names *path where you put write your labels.\nIn Colab we can use the **magic command **to write directly into the file using a cell. Everything under the magic command will be copied\ninto the specified file.\n```plain text\n%%writefile darknet/data/obj.names\nbus\ntruck\n```\nNow we need to modify another file to tell YOLO how many classes to expect, where to find the paths for training and\nvalidation and where to find the file with the name of the labels.\nWe can do this simply using the magic command and the following lines.\n```plain text\n%%writefile darknet/data/obj.data\nclasses = 2\ntrain = darknet/data/train.txt\nvalid = darknet/data/val.txt\nnames = darknet/data/obj.names\nbackup = backup/\n```\nSo to understand it, your train txt file should look like you see in the following picture (similar for validation).\n![](https://miro.medium.com/v2/resize:fit:301/1*m6SmVSb6PtQ2M9wRuUEzNA.png)\nIn which each line indicates where to find the training image.\nThe files we specified though are still empty. So we copy this data from the dataset folder we downloaded into the default one within Darknet.\n```plain text\n!mkdir -p darknet/data/obj\n!cp -r open-images-bus-trucks/images/* darknet/data/obj/\n!cp -r open-images-bus-trucks/yolo_labels/all/{train,val}.txt darknet/data/\n!cp -r open-images-bus-trucks/yolo_labels/all/labels/*.txt darknet/data/obj/\n#add prefix 'darkent/' in front of each row in darkent/data/train.txt\n!sed -i -e 's/^/darknet\\//' darknet/data/train.txt\n!sed -i -e 's/^/darknet\\//' darknet/data/val.txt\n```\nAs we did before we go to download the weights for YOLO. This time we take ***yolov4-tiny*** which is faster than the previous one.\nThen we copy the weights to the appropriate folder inside Darknet.\n```plain text\n!wget - quiet https://github.com/AlexeyAB/darknet/releases/download/darknet_yolo_v4_pre/yolov4-tiny.conv.29\n!cp yolov4-tiny.conv.29 darknet/build/darknet/x64/\n```\nNow let’s rename the configuration file that is in charge of configuring the *yolov4 tiny* architecture.\nAnd after that, we’re going to edit some parameters to set the number of batches the number of classes and other parameters.\n```plain text\n%%bash\ncd darknet\n# create a copy of existing configuration and modify it in place\ncp cfg/yolov4-tiny-custom.cfg cfg/yolov4-tiny-bus-trucks.cfg\n# max_batches to 4000 (since the dataset is small enough)\nsed -i 's/max_batches = 500200/max_batches=4000/' cfg/yolov4-tiny-bus-trucks.cfg\n# number of sub-batches per batch\nsed -i 's/subdivisions=1/subdivisions=16/' cfg/yolov4-tiny-bus-trucks.cfg\n# number of batches after which learning rate is decayed\nsed -i 's/steps=400000,450000/steps=3200,3600/' cfg/yolov4-tiny-bus-trucks.cfg\n# number of classes is 2 as opposed to 80\n# (which is the number of COCO classes)\nsed -i 's/classes=80/classes=2/g' cfg/yolov4-tiny-bus-trucks.cfg\n# in the classification and regression heads,\n# change number of output convolution filters\n# from 255 -> 21 and 57 -> 33, since we have fewer classes\n# we don't need as many filters\nsed -i 's/filters=255/filters=21/g' cfg/yolov4-tiny-bus-trucks.cfg\nsed -i 's/filters=57/filters=33/g' cfg/yolov4-tiny-bus-trucks.cfg\n```\n## **Train the model!**\nNow we are ready, all that remains is to launch the model train\n```plain text\n!./darknet/darknet detector train darknet/data/obj.data ./darknet/cfg/yolov4-tiny-bus-trucks.cfg yolov4-tiny.conv.29 -dont_show -mapLastAt\n```\nThis training in my case took about one hour.\nNow you can run the predictions on your images to get the classes and bounding box.\n```plain text\nfrom torch_snippets import Glob, stem, show, read\n# upload your own images to a folder\nimage_paths = Glob('images-of-trucks-and-busses')\nfor f in image_paths:\n !./darknet detector test \\\n data/obj.data cfg/yolov4-tiny-bus-trucks.cfg\\\n backup/yolov4-tiny-bus-trucks_4000.weights {f}\n !mv predictions.jpg {stem(f)}_pred.jpg\nfor i in Glob('*_pred.jpg'):\n show(read(i, 1), sz=20)\n```\n![](https://miro.medium.com/v2/resize:fit:700/1*A-sUpcgRMDnVKT9tl_vBiA.png)\n# **Final Thoughts**\nAs
