lemmatization and the core operations with SpaCy\n### Mastering NLP with spaCY — Part 1\nNatural Language Processing, or NLP, is a part of AI that focuses on understanding text. It’s about helping machines read, process, and find useful patterns or information within a text, for our apps. [SpaCy](https://spacy.io/) is a library that makes this work easier and faster.\nMany developers today use huge models like ChatGPT or Llama for most NLP tasks. These models are powerful and can do a lot, but they’re often costly and slow. In real-world projects, we need something more focused and quick. This is where spaCy helps a lot.\nNow, spaCy even lets you combine its strengths with large models like ChatGPT through the `spacy-llm` module. It's a great way to get both speed and power.\n### Installing Spacy\nCopy and paste the next commands to install spaCy with pip.\nIn the following cells, substitute the “**&ndash** “ with “-”.\n```plain text\npython -m venv. env source\n.env/bin/activate\npip install -U pip setuptools wheel\npip install -U spacy\n```\nSpaCy doesn’t come with a statistical language model, which is needed to perform operations on a particular language. For each language, there are many models based on the size of the resources used to build the model itself.\nAll the languages supported are listed here: [https://spacy.io/usage/models](https://spacy.io/usage/models)\nYou can download a language model via the command line. In this example, I am downloading a language model for the English language.\n```plain text\npython -m spacy download en_core_web_sm\n```\nAt this point, you are ready to use the model with the load() functionality\n```plain text\nimport spacy nlp = spacy.load(\"en_core_web_sm\")\ndoc = nlp(\"This is a text example I want to analyze\")\n```\n### SpaCy Pipeline\nWhen you load a language model in spaCy, it processes your text through a pipeline that you can customise. This pipeline is made up of various **components**, each handling a specific task. At its core is the tokenizer, which breaks the text into individual tokens (words, punctuation, etc.).\nThe result of this pipeline is a `Doc` object, which serves as the foundation for further analysis. Other components, like the Tagger (for part-of-speech tagging), Parser (for dependency parsing), and NER (for named entity recognition), can be included based on what you want to achieve. We will see what Tagger, Parser and NER mean in the upcoming articles.\n![](https://cdn-images-1.medium.com/max/1600/0*ScOpBwSRb_UB_lkp.png)\nIn order to create a doc object, you can simply do the following\n```plain text\nimport spacy\n```\n```plain text\nnlp = spacy.load(\"en_core_web_md\")\ndoc = nlp(\"My name is Marcello\")\n```\nWe will get familiarity with many more container objects provided by spaCy.\n> The central data structures in spaCy are the Language class, the Vocab and the Doc object.\nBy checking the documentation, you will find the whole list of container objects.\n![](https://cdn-images-1.medium.com/max/1600/0*yb-1SuKBUGtCpuOw.png)\nFrom spaCy documentation\n### Tokenization with spaCy\nIn NLP, the first step in processing text is tokenization. This is crucial because all subsequent NLP tasks rely on working with tokens. Tokens are the smallest meaningful units of text that a sentence can be broken into. Intuitively, you might think of tokens as individual words split by spaces, but it’s not that simple.\nTokenization often depends on statistical patterns, where groups of characters that frequently appear together are treated as single tokens for better analysis.\nYou can play with different tokenizer on this hugging face space: [https://huggingface.co/spaces/Xenova/the-tokenizer-playground](https://huggingface.co/spaces/Xenova/the-tokenizer-playground)\nWhen we apply nlp() to some text in spacy, the text is automatically tokenized. Let’s see an example.\n```plain text\ndoc = nlp(\"My name is Marcello Politi\")\nfor token in doc:\n  print(token.text)\n```\n![](https://cdn-images-1.medium.com/max/1600/0*yEIUrXVz2DbAdNX0.png)\nImage by Author\nFrom the example looks like a simple split made with text.split(“”). So let’s try to tokenize a more complex sentence.\n```plain text\ndoc = nlp(\"I don't like cooking, I prefer eating!!!\")\nfor i, token in enumerate(doc):\n  print(f\"Token {i}:\",token.text)\n```\n![](https://cdn-images-1.medium.com/max/1600/0*Fj23eFkeCVRuftNe.png)\nImage by Author\nSpaCy’s tokenizer is rule-based, meaning it uses linguistic rules and patterns to determine how to split text. It is not based on statistical methods like modern LLMs.\nWhat is interesting is that the rules are customizable; this gives you full control over the tokenization process.\nAlso, spaCy tokenizers are non-destructive, which means that from the token you will be able to recover the original text.\nLet’s see how to **customize the tokenizer. **In order to accomplish this, we just need to define a new rule for our tokenizer, we can do this by using the special ORTH symbol.\n```plain text\nimport spacy\nfrom spacy.symbols import ORTH\n\nnlp = spacy.load(\"en_core_web_sm\")\ndoc = nlp(\"Marcello Politi\")\n\nfor i, token in enumerate(doc):\n  print(f\"Token {i}:\",token.text)\n```\n![](https://cdn-images-1.medium.com/max/1600/1*TcR15R0-POrh1-0HvVzb2w.png)\nImage by Author\nI want to tokenize the word “Marcello” differently.\n```plain text\nspecial_case = [{ORTH:\"Marce\"},{ORTH:\"llo\"}]\nnlp.tokenizer.add_special_case(\"Marcello\", special_case)\ndoc = nlp(\"Marcello Politi\")\n\nfor i, token in enumerate(doc):\n  print(f\"Token {i}:\",token.text)\n```\n![](https://cdn-images-1.medium.com/max/1600/1*wQQQ7t13cOC69yz4RZlZAw.png)\nImage by Author\nIn most cases, the default tokenizer works well, and it’s rare for anyone to need to modify it, usually, only researchers do.\nSplitting text into tokens is easier than splitting a paragraph into sentences. SpaCy is able to accomplish this by using a dependency parser; you can learn more about it in the documentation. But let’s see how this works in practice.\n```plain text\nimport spacy\nnlp = spacy.load(\"en_core_web_sm\")\n\ntext = \"My name is Marcello Politi. I like playing basketball a lot!\"\ndoc = nlp(text)\n\nfor i, sent in enumerate(doc.sents):\n  print(f\"sentence {i}:\", sent.text)\n```\n### Lemmatization with spaCy\nWords/tokens can have different forms. A lemma is the base form of a word. For example, “dance” is the lemma of the words “dancing”, “danced”, “dancer”, “dances”.\nWhen we reduce words to their base form, we are applying lemmatisation.\n![](https://cdn-images-1.medium.com/max/1600/1*jl9mrGOvIt7Xf0XxUGC8iA.png)\nIn SpaCy we can have access to words lemma easily. Check the following code.\n```plain text\nimport spacy\n\nnlp = spacy.load(\"en_core_web_sm\")\ndoc = nlp(\"I like dancing a lot, and then I love eating pasta!\")\nfor token in doc:\n    print(\"Text :\", token.text, \"--> Lemma :\", token.lemma_)\n```\n![](https://cdn-images-1.medium.com/max/1600/1*b3p60y0WTyjPasuBF_UiEQ.png)\nImage by Author\n### Final Thoughts\nWrapping up this first part of this spaCy series, I’ve shared the basics that got me hooked on this tool for NLP.\nWe covered setting up spaCy, loading a language model, and digging into tokenization and lemmatization, the main steps that make text processing feel less like a black box.\nUnlike those massive models like ChatGPT that can feel overkill for smaller projects, spaCy’s lean and fast approach fits the needs of many projects perfectly, especially with the option of also using those big models through spacy-llm when you want extra power!\nIn the next part, I’ll walk you through how I use spaCy’s named entity recognition and dependency parsing to tackle real-world text tasks.
