Now we can start coding! 👨 ‍💻\n<empty-block/>\nI want to instantiate the tiny-llama-1B model. The model and the *model card* can be easily found on [HuggingFace(HF) at this link](https://huggingface.co/TinyLlama/TinyLlama-1.1B-Chat-v1.0).\nModel cards are very important because they help us users understand well how the model works and how to use it. That’s why I always doubt projects on HuggingFace that are not accompanied by good descriptions.\nNow, simply by following what the model card says on HF, I create a function that given an input query generates the answer. I do all this in a new file that I call *model.py*\nThe first time you use the model it will take a long time because it has to be downloaded from Hugging Face.\nKeep in mind that I run this model on CPU because we will be using in free instantiation of AWS which does not provide GPU access.\n```python\n# Install transformers from source - only needed for versions <= v4.34\n# pip install git+https://github.com/huggingface/transformers.git\n# pip install accelerate\n\nimport torch\nfrom transformers import pipeline\n\ndef model_query(query: str):\n    pipe = pipeline(\n        \"text-generation\",\n        model=\"TinyLlama/TinyLlama-1.1B-Chat-v1.0\",\n        torch_dtype=torch.bfloat16,\n        device_map=\"cpu\",\n    )\n\n    # We use the tokenizer's chat template to format each message - see https://huggingface.co/docs/transformers/main/en/chat_templating\n    messages = [\n        {\n            \"role\": \"system\",\n            \"content\": \"You are a friendly chatbot who always funny and interesting. You only reply with the actual answer without repeating my question.\",\n        },\n        {\"role\": \"user\", \"content\": f\"{query}\"},\n    ]\n    prompt = pipe.tokenizer.apply_chat_template(\n        messages, tokenize=False, add_generation_prompt=True\n    )\n    outputs = pipe(\n        prompt,\n        max_new_tokens=256,\n        do_sample=True,\n        temperature=0.7,\n        top_k=50,\n        top_p=0.95,\n    )\n\n    output = outputs[0][\"generated_text\"]\n    return output\n\nif __name__ == \"__main__\":\n    print(model_query(\"tell me a joke about politicians\"))\n```\nPerfect now we have our function. All we have to do is **create a web service** that allows you to use this function via **API**, so anyone can connect and use the model.\nThere are several frameworks to do this, for example, [Flask](https://flask.palletsprojects.com/en/3.0.x/), and [Django](https://www.djangoproject.com/), but here I will use [FastAPI](https://fastapi.tiangolo.com/).\nI create a *main.py* file where I set up the endpoints of my API.\n```python\nfrom fastapi import FastAPI\nfrom model import model_query\nfrom pydantic import BaseModel\n\nclass Query(BaseModel):\n    prompt: str\n\napp = FastAPI()\n\n@app.get(\"/\")\nasync def root():\n    return {\"message\": \"Hello!\"}\n\n@app.post(\"/query\")\nasync def root(query: Query):\n    res = model_query(query=query.prompt)\n    return {\"message\": f\"{res}\"}\n```\nYou see that in this case, the most important API is the one under the \"/query\" route. It expects as input an object of the type Query defined above, basically a JSON object in which a prompt is defined, and returns a JSON containing the model message.\nTo launch the FastAPI service, simply use the following terminal command.\n```css\nuvicorn main:app --reload\n```\nA link will appear that you can click on to access the service we just created in localhost.\n![](https://towardsdatascience.com/wp-content/uploads/2024/01/1rTd-ljJgMVUT8K6eHLXyfg.png)\nFastAPI Service \nIf you click on that link you should see a screen with the messaging set on the main API at route \"/\".\n![](https://towardsdatascience.com/wp-content/uploads/2024/01/1HaWhnGahEX5_NPXzeQbkDQ.png)\n<empty-block/>\nA very useful feature of FastAPI is that typing a *\"/docs\"* next to the URL will show us all the APIs developed, and allow us to use them, without using external services such as [Postman](https://www.postman.com/).\n![](https://towardsdatascience.com/wp-content/uploads/2024/01/1_GbZzSIKV2fC7gzZMDgGIw.png)\n<empty-block/>\nLet’s then try to use the *query API* from this interface, just click on it.
