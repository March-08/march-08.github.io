**Introduction**\nIn my [previous article](https://medium.com/towards-data-science/hands-on-generative-ai-with-gans-using-python-autoencoders-c77232b402fc), we learned about Autoencoders, now let’s continue to talk about Generative AI. By now everyone is talking about it and everyone is excited about the practical applications that have been developed. But we continue to see the foundations of these AIs step by step.\nThere are several Machine Learning models that allow us to build generative AI, to name a few we have Variational Autoencoders (VAE), autoregressive models and even normalizing flow models. In this article, however, we will focus on GANs.\n## **Autoencoders and GANs**\nIn the previous article, we dealt with autoencoders and saw their architecture, their use and implementation in PyTorch.\nIn short, Autoencoders receive an input x, compress it into a vector of smaller size z, called the latent vector, and finally from z reconstruct x in a more or less approximate way.\nIn Autoencoder we have no data generation, but simply an approximate reconstruction of the input. Now imagine that we break the Autoencoder in two and consider only the second part, the part where from the latent vector z the image is reconstructed.\n![](https://miro.medium.com/v2/resize:fit:700/1*kL6Ptt_5-MEqpjCON5tnVQ.png)\nOutput Generation (Image By Author)\nIn this case, we can say that the architecture is generative. In fact, given a vector of numbers as input this creates an image! Essentially this is what a generative AI does. The main difference though with respect to autoencoders is that we know well the probability distribution from which we take the latent vector z. For example, a *Gaussian(0,1)*.\nSo we thus have a way to generate images from random numbers taken from a Gaussian distribution, changing these random numbers will change the images we have in the output.\n![](https://miro.medium.com/v2/resize:fit:700/1*Z6O7bpuh4vEBPECkvncOCA.png)\nGenerative Model (Image By Author)\n## **GANs Architecture**\nThe orange network shown in the previous image can be defined as a G function that given the input z generates the synthetic output *x_cap*, so *x_cap = G(z)*.\nThe network will be initialized with random weights, so it will not initially be able to generate output that looks real, but only images that will contain noise. So we need to do some training to improve the performance of our network.\nSo let’s imagine that we have a human annotator telling us each time whether the output is good or not, whether it looks real or not.\n![](https://miro.medium.com/v2/resize:fit:700/1*x2xGymhzm2ar9WAU3OcEcQ.png)\nTowards GANs (Image By Author)\nObviously, we cannot do network training expecting a person to make continuous judgments about the output. But then what can we do?\nIf you think about it what the annotator does, in this case, is binary classification! And we in Machine Learning are great at developing classifiers. So we can simply train a classifier that we’ll call Discriminator, and we’ll denote with the function D(), which has to be trained to recognize synthetic (fake) images versus real images. So we will feed it both fake images and real images.\nSo this is how our architecture changes.\n![](https://miro.medium.com/v2/resize:fit:700/1*h07aDSTfNQuoPk1a1WhxlQ.png)\nGANs Architecture (Image By Author)\nIn short, the architecture is not too complex. The difficulty comes at the time of having to train these two networks G and D.\nIt is clear that if in training, the two networks have to improve together, they need to find some kind of balance. Because if, for example, D gets too good at distinguishing fake images from real ones before G gets good at generating them, it is quite natural that G will never get better and we will never have our generator ready to be used.\nSo the two networks are said to play an adversarial game in which G must fool D, and D must not be fooled by G.\n## **GANs Objective Function**\nIf we want to be a bit more precise, we can say that D and G have two complementary goals. Let’s suppose we want to generate images.\nWe define by D(x) the probability that x is a real image. Obviously, the discriminator wants to maximize its probability of recognizing real inputs from fake inputs. So we want to maximize D(x) when x is drawn from our distribution of real images.\nIn contrast, the purpose of the generator G is to fool the discriminator. So if *G(z)* is the fake image generated by G, *D(G(z))* is the probability that D will recognize a fake image as real. Then *1-D(G(z))* is the probability that D correctly recognizes a fake image as fake. So G’s goal is to minimize *1-D(G(z))*, since he does want to fool D.\nSo in the end we can sum up this game of maximization and minimization in the formula we find in the original paper (the formula looks a bit more concept but we have seen the concept):\n![](https://miro.medium.com/v2/resize:fit:509/1*Ys5Zuxs8aVwgTCD2qBbWgg.png)\nObjective Function (src: [https://arxiv.org/pdf/1406.2661.pdf](https://arxiv.org/pdf/1406.2661.pdf))\n## **GANs Implementation**\nWe now implement a GAN capable of generating MNIST images automatically.\nAs usual, I will run my code a cloud-based environment Deepnote but you can use Google Colab as well, so even those who don’t have a GPU on their laptop can run this code.\nWe start by going to check whether indeed our hardware has a GPU.\n```python\nimport torch\nprint(torch.__version__)\nprint(\"GPU available: \", torch.cuda.is_available())\ndevice = torch.device(\"cuda:0\") if torch.cuda.is_available() else \"cpu\"\n```\nNow if you’re using Colab you can connect to Google Drive.\n```plain text\nfrom google.colab import drive\ndrive.mount('/content/drive/')\n```\nLet’s import the needed libraries.\n```python\nimport torch.nn as nn\nimport numpy as np\nimport matplotlib.pyplot as plt\n```\nNow we need to create the functions that will define our networks, generator and discriminator.\nThe MNIST images have 784 pixels (since the images are 28x28). So the generator given as input a random z vector of length 20 will have to output a vector of 784 which will be our fake image.\nInstead, the discriminator will receive as input a 28x28 = 784-pixel image, it will have a single neuron in output that will classify the image as true or fake.\n![](https://miro.medium.com/v2/resize:fit:700/1*UR7toURxBSiT1Ol0DGcndQ.png)\n```python\ndef make_generator(\n    input_size:int = 20,\n    num_hidden_layers:int = 1,\n    num_hidden_units:int = 100,\n    num_output_units:int = 784):\n  \n  model = nn.Sequential()\n  for i in range(num_hidden_layers):\n    model.add_module(f'fc_g{i}',\n                     nn.Linear(input_size, num_hidden_units))\n    model.add_module(f'relu_g{i}', nn.LeakyReLU()) \n    input_size = num_hidden_units\n  model.add_module(f'fc_g{num_hidden_layers}',\n                  nn.Linear(input_size, num_output_units))\n  model.add_module('tanh_g', nn.Tanh())\n  return model\n```\nThis function is used to instantiate the generator. Each layer will use a LeakyReLU (a variation of the ReLU, that works best in GANs) as its activation function, except the output is followed by a Hyperbolic Tangent (Tanh) function that results in output a number in the range \\[-1,1\\].\n![](https://miro.medium.com/v2/resize:fit:700/1*pgHYFnIpoYbBuOfZCxx6Mw.png)\n```python\ndef make_discriminator(\n  input_size:int = 784,\n  num_hidden_layers:int = 1,\n  num_hidden_units:int = 100,\n  num_output_units:int = 1\n):\n  model = nn.Sequential()\n  for i in range(num_hidden_layers):\n    model.add_module(f'fc_d{i}',\n                      nn.Linear(input_size, num_hidden_units, bias = False))\n    model.add_module(f'relu_d{i}', nn.LeakyReLU())\n    model.add_module('dropout', nn.Dropout(p = 0.5))\n    input_size = num_hidden_units\n  model.add_module(f'fc_d{num_hidden_layers}',\n                  nn.Linear(input_size, num_output_units))\n  model.add_module('sigmoid', nn.Sigmoid())\n  return model\n```\nInstead, this function defines the discriminator network, which has the special feature of using dropout after hidden layers (in the base case only one hidden layer). The output goes through a sigmoid function since it must give us the probability of being a real or fake image.\nNow we also download the MNIST dataset that we are going to use. The MNIST dataset is in a range \\[0,255\\], but we want it in the range \\[-1,1\\] so that it will be similar to the data generated by the Generator network. So we also apply some preprocessing to do this.\n```python\nimport torchvision\nfrom torchvision import transforms\n\nimage_path = './'\ntransform = transforms.Compose([\n    transforms.ToTensor(),\n    transforms.Normalize(mean = (0.5), std = (0.5)),\n])\n\nmnist_dataset = torchvision.datasets.MNIST(\n    root = image_path, train = True,\n    transform = transform, download = True\n)\n```\nNow we come to the most important part. We need to create the functions that define the training of our network. We have already said that we should pull the discriminator separately from the generator, so we will have 2 functions.\n```python\nloss_fn = nn.BCELoss()\n\n## train discriminator\ndef d_train(x):\n  disc_model.zero_grad()\n  \n  #train discriminator with a real batch\n  batch_size = x.size(0)\n  x = x.view(batch_size, -1).to(device) #reshape input\n  d_labels_real = torch.ones(batch_size, 1, device = device)\n  d_proba_real = disc_model(x)\n  d_loss_real = loss_fn(d_proba_real, d_labels_real)\n\n  #train discriminator with a fake batch\n  input_z = create_noise(batch_size, z_size, mode_z).to(device)\n  g_output = gen_model(input_z)\n  d_proba_fake = disc_model(g_output)\n  d_labels_fake = torch.zeros(batch_size, 1, device = device)\n  d_loss_fake = loss_fn(d_proba_fake, d_labels_fake)\n\n  #gradient backprop & otimizeonlys Disc\n  d_loss = d_loss_real + d_loss_fake\n  d_loss.backward()\n  d_optimizer.step()\n\n  return d_loss.data.item(), d_proba_real.detach(), d_proba_fake.detach()\n```\nThe discriminator will be trained both on the fake data and on real data. When we train it on real data the labels will always be *“real” = 1*. So we create a vector of 1 with *d_labels_real = torch.ones(batch_size, 1, device = device)*. Then we feed the input x to the model and calculate the loss using *Binary Cross Entropy*.\nWe do the same thing by feeding fake data. Here the labels will all be zero, *d_labels_fake = torch.zeros(batch_size, 1, device = device)*. The input instead will be the fake data, that is, the output of the generator *g_output = gen_model(input_z)*. And we calculate the loss in the same way.\nThe final loss will be the sum of the two losses.\n```python\n## train generator\ndef g_train(x):\n  gen_model.zero_grad()\n  batch_size = x.size(0)\n  input_z = create_noise(batch_size, z_size, mode_z).to(device)\n  g_labels_real = torch.ones(batch_size, 1, device = device)\n\n  g_output = gen_model(input_z)\n  d_proba_fake = disc_model(g_output)\n  g_loss = loss_fn(d_proba_fake, g_labels_real)\n\n  #gradient backprop & otpimize Gen\n  g_loss.backward()\n  g_optimizer.step()\n  return g_loss.data.item()\n```\nAs for the generator train function, the implementation is slightly different. The generator takes as input the output of the discriminator since it has to see if D has figured out whether it is a fake or real image. And based on that it calculates its loss.\nNow we can initialize our two networks.\n```python\nimage_size = (28,28)\nz_size = 20\ngen_hidden_layers = 1\ngen_hidden_size = 100\ndisc_hidden_layers = 1\ndisc_hidden_size = 100\n\ngen_model = make_generator(\n    input_size = z_size,\n    num_hidden_layers = gen_hidden_layers,\n    num_hidden_units = gen_hidden_size,\n    num_output_units = np.prod(image_size)\n)\n\ndisc_model = make_discriminator(\n    input_size = np.prod(image_size),\n    num_hidden_layers = disc_hidden_layers,\n    num_hidden_units = disc_hidden_size\n)\n\ngen_model = gen_model.to(device)\ndisc_model = disc_model.to(device)\n\nloss_fn = nn.BCELoss()\ng_optimizer = torch.optim.Adam(gen_model.parameters())\nd_optimizer = torch.optim.Adam(disc_model.parameters())\n```\nLet’s define a function to create network-generated samples, so as we go along we can see how the fake images improve as the training epochs increase.\n```python\nfixed_z = create_noise(batch_size, z_size, mode_z).to(device)\ndef create_samples(g_model, input_z):\n  g_output = g_model(input_z)\n  images = torch.reshape(g_output, (batch_size, image_size[0], image_size[1]))\n  return (images+1)/2.0\n```\nNow we can finally train the net! We save the losses each time in a list so we can plot them later.\n```python\nfrom tqdm import  tqdm\n\nepoch_samples = []\nall_d_losses = []\nall_g_losses = []\nall_d_real = []\nall_d_fake = []\nnum_epochs = 100\n\nfor epoch in tqdm(range(1, num_epochs+1), total = num_epochs):\n  d_losses, g_losses = [], []\n  d_vals_real, d_vals_fake = [], []\n  for i, (x,_) in enumerate(dataloader):\n    d_loss, d_proba_real, d_proba_dake = d_train(x)\n    d_losses.append(d_loss)\n    g_losses.append(g_train(x))\n    d_vals_real.append(d_proba_real.mean().cpu())\n    d_vals_fake.append(d_proba_fake.mean().cpu())\n\n  all_d_losses.append(torch.tensor(d_losses).mean())\n  all_g_losses.append(torch.tensor(g_losses).mean())\n  all_d_real.append(torch.tensor(d_vals_real).mean())\n  all_d_fake.append(torch.tensor(d_vals_fake).mean())\n  print(f'Epoch {epoch:03d} | Avg Losses >>'\n    f'G/D {all_g_losses[-1]:.4f}/{all_d_losses[-1]:.4f}'\n    f' [D-Real: {all_d_real[-1]:.4f}'\n    f' D-Fake: {all_d_fake[-1]:.4f}]')\n  \n  epoch_samples.append(\n      create_samples(gen_model, fixed_z).detach().cpu().numpy()\n  )\n```\nThe training should take about an hour, depending on the hardware cha you used certainly. But in the end, you can print out your fake data and have something like this.\n```python\nselected_epochs = [1, 2, 4, 10, 50, 100]\nfig = plt.figure(figsize = (10,14))\nfor i,e in enumerate(selected_epochs):\n  for j in range(5):\n    ax = fig.add_subplot(6,5, i*5+j+1)\n    ax.set_xticks([])\n    ax.set_yticks([])\n    if j == 0:\n      ax.text(\n          -0.06, 0.5, f'Epoch {e}',\n          rotation = 90, size = 18, color = 'red',\n          horizontalalignment = 'right',\n          verticalalignment = 'center',\n          transform = ax.transAxes\n      )\n      print(e-1,j)\n    image = epoch_samples[e-1][j]\n    ax.imshow(image, cmap = 'gray_r')\nplt.show()\n```\nIn my case, I trained for a few epochs so the results are not great, but you are beginning to get a glimpse that the network was learning to generate MNIST-like images.\n![](https://miro.medium.com/v2/resize:fit:700/1*O7wm9F0lbJjI_TzQQUprsA.png)\n# **Final Thoughts**\nIn this article, we looked at how the architecture of GANs in more detail.
