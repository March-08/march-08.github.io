extraction with SpaCy\n### Mastering NLP with spaCy — Part 3\n### Introduction\nIt is important to understand how to use spaCy rules to identify patterns within some text. There are entities like times, dates, IBANs and emails that follow a strict structure, so it is possible to identify them with deterministic rules, for example, by using **regular expressions** ( **regexes**).\nSpaCy simplifies the usage of regexes by making them more human-readable, so instead of weird symbols, you will use actual descriptions using the Matcher class.\n### Token-based matching\nA regex is a sequence of characters that specifies a search pattern. There is a Python built-in library to work with regexes called **re**: [https://docs.python.org/3/library/re.html](https://docs.python.org/3/library/re.html)\nLet’s see an example.\n```plain text\n\"Marcello Politi\"\n\"Marcello   Politi\"\n\"Marcello Danilo Politi\"\n\nreg = r\"Marcello\\s(Danilo\\a)?Politi\"\n```\nThe problem with regexes, and the reason why many programmers don’t love them, is that they’re difficult to read. This is why spaCy provides a clean and production-level alternative with the **Matcher class**.\nLet’s import the class and see how we can use it. (I will explain what Span is later).\n```plain text\nimport spacy\nfrom spacy.matcher import Matcher\nfrom spacy.tokens import Span\nnlp = spacy.load(\"en_core_web_sm\")\n```\nNow we can define a pattern that matches some morning greetings, and we label this pattern “morningGreeting”. Defining a pattern with Matcher is straightforward. In this pattern, we expect a word that, when converted to lower case, matches the word “good”, then the same for “morning”, and then we accept so punctuation at the end.\n```plain text\nmatcher = Matcher(nlp.vocab)\npattern = [\n    {\"LOWER\": \"good\"},\n    {\"LOWER\": \"morning\"},\n    {\"IS_PUNCT\": True},\n]\nmatcher.add(\"monrningGreeting\", [pattern])\n```\nA Span is a singular sentence, so the Matcher can find the starting and ending point of multiple spans that we iterate over with a for loop.\nWe add all the spans in a list and assign the list to the doc.spans\\[“sc”\\]. Then we can use displacy to visualise the span\n```plain text\ndoc = nlp(\"Good morning, My name is Marcello Politi!\")\nmatches = matcher(doc)\nspans = []\n\nfor match_id, start, end in matches:\n  spans.append(\n      Span(doc, start, end, nlp.vocab.strings[match_id])\n  )\n\ndoc.spans[\"sc\"] = spans\n```\n```plain text\nfrom spacy import displacy\n\ndisplacy.render(doc, style = \"span\")\n```\n![](https://cdn-images-1.medium.com/max/1600/0*OTlY9P4-sj7AvK7_.png)\nImage by Author\n**A Matcher object accepts more than one pattern at a time!**\nLet’s define a morningGreeting and a eveningGreeting.\n```plain text\npattern1 = [\n    {\"LOWER\": \"good\"},\n    {\"LOWER\": \"morning\"},\n    {\"IS_PUNCT\": True},\n]\n\npattern2 = [\n    {\"LOWER\": \"good\"},\n    {\"LOWER\": \"evening\"},\n    {\"IS_PUNCT\": True},\n]\n```\nThen we add these patterns to the Matcher.\n```plain text\ndoc = nlp(\"Good morning, I want to attend the lecture. I will then say good evening!\")\nmatcher = Matcher(nlp.vocab)\n\nmatcher.add(\"morningGreetings\", [pattern1])\nmatcher.add(\"eveningGreetings\", [pattern2])\n\nmatches = matcher(doc)\n```\nAs before, we iterate over the spans and display them.\n```plain text\nspans = []\n\nfor match_id, start, end in matches:\n  spans.append(\n      Span(doc, start, end, nlp.vocab.strings[match_id])\n  )\n\ndoc.spans[\"sc\"] = spans\n```\n```plain text\nfrom spacy import displacy\n\ndisplacy.render(doc, style = \"span\")\n```\n![](https://cdn-images-1.medium.com/max/1600/0*6u8ojx27jbPVrPzL.png)\nImage by Author\nThe syntax supported by spaCy is huge. Here I report some of the most common patterns.\n### Text-based attributes\n![](https://cdn-images-1.medium.com/max/1600/1*9se-uFxX-OHdAMrrtuQ0mA.png)\nImage by Author\n### Linguistic features\n![](https://cdn-images-1.medium.com/max/1600/1*H7uhoyJsrdTFEL7ziXC7xQ.png)\nImage by Author\n### Boolean flags\n![](https://cdn-images-1.medium.com/max/1600/1*sK5rSPpQNyX0LhAcVnZSCw.png)\nImage By Author\n### Operators\nUsed to repeat or make patterns optional:\n![](https://cdn-images-1.medium.com/max/1600/1*Ma4lkfuPCm0135PsIPKpVw.png)\nImage by Author\nExample:\nWhat is a pattern that matches a string like: *“I have 2 red apples”, “We bought 5 green bananas”, or “They found 3 ripe oranges”* ?\n**Pattern Requirements:**\n- Subject pronoun (e.g., “I”, “we”, “they”)\n- A verb (e.g., “have”, “bought”, “found”)\n- A number (digit or written, like “2”, “five”)\n- An optional adjective (e.g., “red”, “ripe”)\n- A plural noun (fruit, for example)\n```plain text\npattern = [\n    {\"POS\": \"PRON\"},                               # Subject pronoun: I, we, they\n    {\"POS\": \"VERB\"},                               # Verb: have, bought, found\n    {\"LIKE_NUM\": True},                            # Number: 2, five\n    {\"POS\": \"ADJ\", \"OP\": \"?\"},                     # Optional adjective: red, ripe\n    {\"POS\": \"NOUN\", \"TAG\": \"NNS\"}                  # Plural noun: apples, bananas\n]\n```\n### Patterns with PhraseMatcher\nWe we work in a vertical domain, like medical or scientific, we usually have a set of words that spaCy might not be aware of, and we want to find them in some text.\nThe PhraseMatcher class is the spaCy solution for **comparing text against long dictionaries**. The usage is quite similar to the Matcher class, but in addition, we need to define the list of important terms we want to track. Let’s start with the imports.\n```plain text\nimport spacy\nfrom spacy.tokens import Span\nfrom spacy.matcher import PhraseMatcher\nfrom spacy import displacy\nnlp = spacy.load(\"en_core_web_sm\")\n```\nNow we define our matcher and our list of words, and tell Spacy to create a pattern just to recognise that list. Here, I want to identify the names of tech leaders and places.\n```plain text\nterms = [\"Sundar Pichai\", \"Tim Cook\", \"Silicon Valley\"]\nmatcher = PhraseMatcher(nlp.vocab)\npatterns = [nlp.make_doc(text) for text in terms]\nmatcher.add(\"TechLeadersAndPlaces\", patterns)\n```\nFinally, check the matches.\n```plain text\ndoc = nlp(\"Tech CEOs like Sundar Pichai and Tim Cook met in Silicon Valley to discdoc = nlp(\"Tech CEOs like Sundar Pichai and Tim Cook met in Silicon Valley to discuss AI regulation.\")\nmatches = matcher(doc)\nspans= []\n\nfor match_id, start, end in matches:\n  pattern_name = nlp.vocab.strings[match_id]\n  spans.append(Span(doc, start, end, pattern_name))\n\ndoc.spans[\"sc\"] = spans\ndisplacy.render(doc, style = \"span\")\n```\n![](https://cdn-images-1.medium.com/max/1600/0*jJcYtIGkQkKgxjJR.png)\nImage by Author\nWe can enhance the capabilities of the PhraseMatcher by adding some attributes. For example, if we need to cach IP addresses in a text, maybe in some logs, we cannot write all the possible combinations of IP addresses, that would be crazy. But we can ask Spacy to catch the shape of some IP strings, and check for the same shape in a text.\n```plain text\nmatcher = PhraseMatcher(nlp.vocab, attr= \"SHAPE\")\n\nips  = [\"127.0.0.0\", \"127.256.0.0\"]\npatterns = [nlp.make_doc(ip) for ip in ips]\nmatcher.add(\"IP-pattern\", patterns)\n```\n```plain text\ndoc = nlp(\"This fastAPI server can run on 192.1.1.1 or on 192.170.1.1\")\nmatches = matcher(doc)\nspans= []\n\nfor match_id, start, end in matches:\n  pattern_name = nlp.vocab.strings[match_id]\n  spans.append(Span(doc, start, end, pattern_name))\n\ndoc.spans[\"sc\"] = spans\ndisplacy.render(doc, style = \"span\")\n```\n![](https://cdn-images-1.medium.com/max/1600/0*EuQsakgQ6DocrA70.png)\nImage by Author\n### IBAN Extraction\nThe IBAN is an important information that we often need to extract when working in the financial fields, for example if are analysing invoices or transactions. But how can we do that?\nEach IBAN has a fixed international number format, starting with two letters to identify the country.\nWe are sure that each IBAN starts with two capital letters XX followed by at least two digits dd. So we can write a pattern to identify this first part of the IBAN.\n```plain text\n{\"SHAPE\":\"XXdd\"}\n```\nIt’s not done yet. For the rest of the block we might have from 1 to 4 digits that we can express with the symbol “\\\\d\\{1,4\\}”.\n```plain text\n{\"TEXT\":{\"REGEX:\"\\d{1,4\"}}\n```\nWe can have one or more of these blocks, so we can use the “+” operator to identify all of them.\n```plain text\n{\"TEXT\":{\"REGEX\":\"\\d{1,4}, \"OP\":\"+\"}\n```\nNow we can combine the shape with the blocks identification.\n```plain text\npattern =[\n   {\"SHAPE\":\"XXdd\"},\n   {\"TEXT\":{\"REGEX\":\"\\d{1,4}\", \"OP\":\"+\"}\n   ]\n\nmatcher = Matcher(nlp.vocab)\nmatcher.add(\"IBAN\", [patttern])\n```\nNow let’s use this!\n```plain text\ntext = \"Please transfer the money to the following account: DE44 5001 0517 5407 3249 31 by Monday.\"\ndoc = nlp(text)\n\nmatches = matcher(doc)\nspans = []\n\nfor match_id, start, end in matches:\n    span = Span(doc, start, end, label=nlp.vocab.strings[match_id])\n    spans.append(span)\n\ndoc.spans[\"sc\"] = spans\ndisplacy.render(doc, style=\"span\")\n```\n![](https://cdn-images-1.medium.com/max/1600/0*tsA3zUFLjISU8Kz5.png)\nImage by Author\n### Final Thoughts\nI hope this article helped you to see how much we can do in NLP without always using huge models.
