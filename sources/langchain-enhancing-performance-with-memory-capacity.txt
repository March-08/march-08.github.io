Now I would like to focus on a key aspect, how to manage memory in intelligent chatbots.\nChatbots or agents also need an information storage mechanism, which can take different forms and perform different functions. **Implementing a memory system in chatbots not only helps make them more clever, but also more natural and useful for users**.\nFortunately, LangChain provides APIs that make it easy for developers to implement memory in their applications. In this article, we will explore this aspect in more detail.\n## **Use Memory in LangChain**\nA best practice when developing chatbots is to** save all the interactions the chatbot has with the user.** This is because the** state of the LLM can change depending on the past conversation**, in fact, the LLM to the same question from 2 users will also answer differently because they have a different past conversation with the chatbot and therefore it is in a different state.\nSo what the chatbot memory creates is nothing more than a list of old messages, which are fed back to it before a new question is asked. Of course, the **LLM have limited context, so you have to be a little creative and choose how to feed this history back to the LLM**. The most common methods are to return a summary of the old messages or return only the N latest messages that are probably the most informative.\n## **Start with the basics with ChatMessageHistory**\nThis is the main class that allows us to manage the messages that occur between the chatbot (AI) and the user (Human). This class provides two main methods which are The following.\n- **add_user_message: **allows us to add a message into the chatbot’s memory and tag the message as “user”\n- **add_ai_message: **allows us to add a message into the chatbot’s memory and tag the message as “AI”\n```plain text\n!pip install langchain\nfrom langchain.memory import ChatMessageHistory\n\nhistory = ChatMessageHistory()\n\nhistory.add_user_message(\"Hi!\")\nhistory.add_ai_message(\"Hey, how can I help you today?\")\n\n#print messages\nhistory.messages\n```\nThis class allows you to do various things, but in its simplest use, you can look at it as saving various messages to a list from time to time. Then you can also review all the messages you have added simply by iterating over the history in the following way.\n```plain text\nfor message in history.messages:\n    print(message.content)\n```\n## **Advanced Memory with ConversationBufferMemory**\nThe **ConversationBufferMemory** class behaves somewhat like the ChatMessageHistory class with respect to the message store, though it **provides clever methods to retrieve old messages**.\nFor example, we can retrieve old messages, as a list of messages or as one big string depending on what we need. If we want to ask the LLM to make a summary of the past conversation it might be useful to have the past conversation as one big string. If we want to do a detailed analysis of the past instead, we can read one message at a time by extracting a list.\nAlso with the ConversationBufferMemory class, we can add messages to the history using the add_user_message and add_user_message methods.\nThe load_memory_variables method on the other hand is used to extract old messages in list or dictionary form depending on what is specified, let’s see an example.\n```plain text\nfrom langchain.memory import ConversationBufferMemory\n\nmemory = ConversationBufferMemory()\n\nmemory.chat_memory.add_user_message(\"Hi, how are you?\")\nmemory.chat_memory.add_ai_message(\"hey,I am fine! How are you?\")\n\nmemory_variables = memory.load_memory_variables({})\nprint(memory_variables['history'])\n```\n## **Manage Memory in Multiple Conversations**\nWe have seen toy examples of how to manage memory by saving and retrieving messages. **But in a real-world application you will probably need to manage the memory of several conversations**. LangChain allows you to manage this case as well with the use of what are called **chains.**\n**A chain is nothing more than a workflow of various simple or complex steps that allow you to achieve a certain goal.**\n**For example, an LLM that looks up a piece of information on Wikipedia because it does not know how to answer a certain question is a chain.**\nTo handle various conversations, it is enough to associate a ConversationBufferMemory with each chain that is created with an instantiation of the ConversationChain class.\nThis way when the predict method of the model is called, all the steps of the chain are run so the model will read the past messages of the conversation.\nLet’s look at a simple example.\n```plain text\nfrom langchain.llms import OpenAI\nfrom langchain.chains import ConversationChain\nfrom langchain.memory import ConversationBufferMemory\n\n\nmemory = ConversationBufferMemory()\nconversation = ConversationChain(llm= OpenAI(), memory=memory)\n\nconversation.predict(input=\"Hey! how are you?\")\n```\n# **Final Thoughts**\nIn conclusion,** memory is a critical component of a chatbot,** and LangChain provides several frameworks and tools to manage memory effectively. Through the use of classes such as** ChatMessageHistory and ConversationBufferMemory, you can capture and store user interactions with the AI**, and use this information to guide future AI responses.
